# B2Bデモ実装計画：愛管株式会社データ活用

## 背景

ikimon.lifeのB2Bデモページで、架空データではなく**愛管株式会社の実データ**（シードデータ）を使って「リアルなプラットフォーム体験」を提供する。

---

## データ公開可否判定

### 既存データの構造

| データ | 内容 | 件数 |
|:---|:---|:---|
| `sites/aikan_hq/meta.json` | サイト名、オーナー名、説明文 | 1件 |
| `sites/aikan_hq/boundary.geojson` | 浜松エリアのPolygon座標 | 1件 |
| `observations/2025-*.json` | 観察データ（種名、GPS、ユーザー名） | 200件超 |
| `corporations.json` | 法人アカウント | 1件（Ikimon Labのみ） |

### 🟢 見せてOK

| データ項目 | 理由 |
|:---|:---|
| **サイト名**（愛管株式会社 本社ビオトープ） | 30by30参加を公表しており、むしろPR効果あり |
| **サイト説明文** | 公開情報ベース。「30by30アライアンス参加企業」は既に公知 |
| **観察された生物種名**（和名+学名） | 生物種の存在は公益的情報。iNaturalistでも全データ公開 |
| **BISスコア & 統計サマリー** | レポートの中核。集約された数値は個人特定リスクゼロ |
| **データ品質分布**（Research Grade比率等） | 集約統計。プラットフォームの信頼性を証明する |
| **月別観察トレンドグラフ** | 集約データ。モニタリングの継続性を証明 |
| **分類群構成**（昆虫○種、鳥類○種...） | 集約統計。サイトの生態系の豊かさを示す |

### 🟡 加工して見せる（匿名化）

| データ項目 | 対応 |
|:---|:---|
| **ユーザー名** | シードデータだが、デモ上は `Observer #1` 等に匿名化表示 |
| **個別観察の正確なGPS** | サイトの**概要マップ**は表示OK。個別座標はサイトエリア内のドットヒートマップとして集約表示 |
| **ユーザーアバター** | pravatar.ccのランダム画像のため問題なし。ただしデモでは表示しない |

### 🔴 見せない

| データ項目 | 理由 |
|:---|:---|
| **ユーザーID/内部キー** | システム内部情報 |
| **個人名と紐付いた観察詳細** | デモでは「誰が」ではなく「何が観察されたか」に焦点 |
| **ノート/メモの生テキスト** | 内部コミュニケーション的な内容を含む可能性 |
| **corporations.json の内部構造** | 法人プランの内部設定は非公開 |

## サイトIDの変更

> [!IMPORTANT]
> ユーザー指示に基づき、内部サイトIDを `aikan_hq` → `i-kan_hq` に変更する。
> ただし、**既存の全データとの整合性を維持するため、デモ用の参照ではメタ情報のownerフィールドのみ変更** し、データファイル名のリネームは行わない。

**方針**: デモページは `aikan_hq` のデータを読み、表示名のみ「愛管株式会社 (I-KAN Co., Ltd.)」として出力する。

## 提案する変更

### デモレポートページ `/demo/report`

#### [NEW] [demo_report.php](file:///g:/その他のパソコン/マイ ノートパソコン/antigravity/ikimon/ikimon.life/upload_package/public_html/demo/report.php)

既存の `generate_site_report.php` ロジックを流用し、**ログイン不要・読み取り専用**で愛管のレポートを表示するデモ専用ページ。

**表示するセクション：**
1. **ヘッダー**: 「愛管株式会社 本社ビオトープ — サイトレポート（デモ）」
2. **概要**: BISスコア（ゲージUI）、総観察数、確認種数、Research Grade比率
3. **種一覧テーブル**: 和名、学名、観察回数、データ品質（ユーザー名なし）
4. **月別トレンドチャート**: 観察数の時系列推移
5. **分類群構成**: 昆虫/鳥類/植物/魚類/哺乳類の円グラフ
6. **レッドリスト照合結果**: 該当種のハイライト
7. **TNFD LEAP対応マッピング**: 各LEAPステップとの対応表
8. **デモバナー**: 上部に常時「📊 これはデモレポートです。あなたのサイトでも同じレポートを生成できます → 無料トライアル」

**技術:**
- ルート: `/demo/report.php` （新規）
- データ: `aikan_hq` のサイトデータ＆全observationをサーバーサイドで読み込み
- 匿名化: user_name, user_id, noteフィールドはレンダリング時にフィルタ（PHPサーバーサイドで除外）
- 表示名: meta.json の owner ではなく、ハードコードで「愛管株式会社 (I-KAN Co., Ltd.)」
- スコア計算: 既存の `generate_site_report.php` のロジックをインポート

#### [NEW] [demo_index.php](file:///g:/その他のパソコン/マイ ノートパソコン/antigravity/ikimon/ikimon.life/upload_package/public_html/demo/index.php)

デモのランディングページ。「プラットフォームを体験する」導線。

## 検証計画

### 自動テスト
```powershell
# PHP構文チェック
php -l upload_package/public_html/demo/report.php
php -l upload_package/public_html/demo/index.php
```

### ブラウザ手動検証
1. `http://localhost:8899/demo/` にアクセス → ランディングページが表示される
2. 「レポートを見る」ボタンをクリック → `/demo/report.php` に遷移
3. レポートページで以下を確認:
   - BISスコアが表示される
   - 種一覧にユーザー名が一切表示されない ⚠️
   - 個別GPSが表示されない ⚠️
   - 月別トレンドチャートが表示される
   - デモバナーが上部と下部に常時表示される
   - 「無料トライアル」CTAが見える
4. モバイルビュー（375px幅）でレスポンシブ確認
