## 1. 見落としていた最大の問題: Gemini Nano の同定精度

**問題**: Nano は「科レベル」（例: シジミチョウ科）止まり。種レベル同定ができない。
**影響**: 日本にシジミチョウ科だけで80種以上。希少種か普通種かも不明。データとしての価値が低い。

**解決**: AI分業アーキテクチャ
- TFLite 種同定モデル（Apache 2.0）→ 種レベル精度
- Gemini Nano → 環境分析・画像品質評価に専念
- Google Perch v2（Apache 2.0）→ 鳥音声同定（オンデバイス）
- Fusion Engine → 3系統を統合して最終判定

## 2. Nano トークン制限（~4K）での全詰め込みは非現実的

**問題**: 種同定 + 画像品質 + 環境分析 + カメラ推奨を1回の推論で返すのは無理。
**解決**: 推論を分割
- 推論1（2秒毎）: TFLite → 種同定
- 推論2（10秒毎）: Nano → 環境分析 + カメラ推奨
- 推論3（3秒毎）: Perch v2 → 鳥音声

## 3. Expert Review グルーピング

**問題**: GPS 50m + 時間5分 + 綱レベル一致では粗すぎる。同一個体判別は野外では不可能。
**解決**: 個体判別はExpertに委ねる。グルーピングは「文脈セット」として捉え、場所+時間+分類で十分。

## 4. Camera2 レンズ自動切替 UX

**問題**: 勝手にズームされると鳥を見失う。
**解決**: 3段階設計（デフォルト: Suggestion Only → PiP Preview → Full Auto）。Full Auto はデフォルトにしない。

## 5. バッテリー現実

**問題**: 全センサー同時稼働で「8-12時間」は嘘。実際は2.7時間。
**解決**:
- 常時スキャン: 3-4時間
- 待ち伏せモード（音響のみ常時 + トリガー起動）: 10-12時間
- 加速度計でスキャン間隔を自動制御

## 6. TFLite カスタムモデル — 最重要の未着手項目

**結論**: Nano を種同定に使うのは「ナイフでネジを回す」ようなもの。専用TFLiteモデルを最優先導入。

## 7. データ構造 — DarwinCore 互換

**問題**: Detection データクラスに taxon_rank がない。Nano が返しているのに捨てている。
**解決**: Detection に taxonRank, confidenceScore, aiModel, environment を追加。
既存の CanonicalStore（6層永続化）、DwcExportAdapter（23フィールドCSV）は対応済み。

## 8. プライバシー設計

**3つの穴**:
1. GPS軌跡（3秒間隔900ポイント）→ 同期時に間引き + 同意画面
2. 音声に人の声混入 → VAD で検出・フラグ
3. 希少種のローカルデータ → PrivacyFilter リストをアプリにキャッシュ

**APPI対応**: 初回起動時に GPS・音声・GBIF国外転送の同意を取得。

## 9. 活用すべきセンサー

**やる**: 加速度計（スキャン間隔制御）、磁力計（コンパス方位）、気圧計（天気予測）
**やらない**: AR Core（重すぎ）、Google Lens API（Nano+TFLiteでカバー可）

## 10. iOS 対応 / KMP

**結論**: 今考えるな。SYNC_ENABLED = false のまま。1プラットフォームで完璧にしてから。
