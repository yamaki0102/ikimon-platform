## A1. 適用法令と根拠条文

### A1.1 APPI（個人情報の保護に関する法律）

| 論点 | 条文 | ikimon.lifeの対応 |
|------|------|------------------|
| 利用目的の特定・通知 | 第17条・第21条 | 利用目的を「生物多様性データの収集・標準化・公開・研究利用」と特定し、利用規約およびアプリ初回起動時に明示 |
| 位置情報の取り扱い | 第2条第1項（個人情報）、第16条第3項（個人データ）、第2条第7項（個人関連情報） | サーバー上のGPS座標はrecordedBy・時刻と結合するため、**個人データとして扱う**（個人関連情報に留まるとする甘い解釈は取らない）。raw位置ログ（通勤ルート等）はサーバーに送信しない。候補イベントのGPS座標のみ送信し、coordinateUncertaintyInMetersで精度を記録 |
| 第三者提供（GBIF等への公開） | 第27条 | DwC-A export時にPrivacyAccess Layerのcoordinate丸めを適用。希少種は自動fuzzing。ユーザーの同意は利用規約でCC-BY 4.0ライセンスへの許諾として取得 |
| 学術研究例外 | 第18条第3項第5号（利用目的変更の例外）、第27条第1項第5号（第三者提供の例外） | ikimon.lifeは私企業であり、単独では「学術研究機関等」に該当しない。大学・博物館との共同研究の場合のみ、共同研究の範囲で学術例外を主張可能。それ以外は原則として同意ベースで運用 |
| 要配慮個人情報 | 第20条第2項 | 生物データ自体は要配慮に該当しないが、顔写真（車載映像等に映り込む可能性）は慎重に扱う。端末側で自動マスキングし、サーバーには送信しない |
| 保有個人データの開示・訂正・削除 | 第33条〜第35条 | ユーザーは自身のデータの開示・訂正・削除を要求可能。削除要求への対応はA3参照 |
| 安全管理措置 | 第23条 | サーバーはXserver VPS（日本国内）。SSL/TLS暗号化。アクセスログ記録。バックアップ |

### A1.2 GDPR適用条件

| 条件 | ikimon.lifeでの該当性 |
|------|---------------------|
| EU/EEA居住者のデータを処理するか | 現時点ではターゲットユーザーは日本国内。ただしGBIFへの公開データは全世界からアクセス可能 |
| EU/EEA居住者に対してサービスを提供するか | 現時点では意図的な提供なし。多言語対応時に再評価が必要 |
| **現時点の判断** | GDPR非適用。ただしEU/EEA向けサービス展開時にDPIA（Data Protection Impact Assessment）を実施する。GBIFへのexportは非個人データ（座標丸め＋匿名化済み）なので、個人データの国外移転には該当しない。**注意：「座標丸め＝匿名加工情報」ではない。APPI上の匿名加工情報（第2条第6項）は厳格な加工基準を満たす必要がある。ikimon.lifeのGBIF exportは「非個人データへの加工」として整理する** |

## A2. データフローのAPPI整理（委託・共同利用・第三者提供）

補助的なGDPR語彙ではなく、APPIの枠組みで整理する。

| フロー | APPI上の整理 | 根拠条文 | 対応 |
|--------|-------------|---------|------|
| ユーザー→ikimon.life | 本人同意に基づく取得 | 第17条、第21条 | 利用規約で利用目的を明示。同意を取得 |
| ikimon.life→Xserver VPS | **委託**（第三者提供に該当しない） | 第27条第5項第1号 | Xserverへの安全管理措置の監督義務（第25条） |
| ikimon.life→Google Gemini API | **委託**（テキストメタデータのみ送信。個人データは含まない設計） | 第27条第5項第1号 | Googleのデータ処理契約。raw個人情報（位置ログ、音声、画像）は送信しない |
| ikimon.life→GBIF | 座標丸め＋recordedBy匿名化により**非個人データ**に加工した上で提供 | 個人データではないためAPPI第27条の制限対象外 | **注意：「匿名加工情報」（第2条第6項）ではなく「非個人データへの加工」。匿名加工情報の厳格な加工基準は適用しない代わりに、元の個人データとの照合が容易でないレベルまで加工する** |
| ikimon.life↔企業（OECM実証） | **共同利用** | 第27条第5項第3号 | 共同利用する旨、項目、範囲、利用目的、責任者を本人に通知。個別契約で定義 |
| ikimon.life↔大学・博物館（共同研究） | **共同利用**＋学術例外の部分適用 | 第27条第5項第3号＋第18条第3項第5号 | 学術例外は大学側の処理にのみ適用。ikimon.life側は同意ベース |

**注記：** ikimon.lifeは私企業であり、単独では「学術研究機関等」（第16条第8項）に該当しない。学術例外の適用は共同研究の枠組み内でのみ。

### 参考：将来のEU展開時に追加で整理する論点

現時点ではAPPIを主法令として設計し、本AppendixでもAPPIの概念で統一する。EU/EEA向け展開を開始する時点でのみ、GDPR上の役割整理、国外移転、DPIA要否を別紙で追加する。

## A3. データ種別ごとの保持・削除ポリシー

| データ種別 | 保持期間 | 削除要求への対応 | 法的根拠 |
|-----------|---------|---------------|---------|
| **Tier 2以上のoccurrence/identification** | 無期限（科学データの継続性） | 匿名化して保持。recordedBy/identifiedByを匿名IDに置換。座標は既に丸め済み。削除ではなく匿名化で対応 | 第35条第1項ただし書き：正当な事由がある場合の削除拒否。科学データの公共性を根拠とするが、法的に確定していないため法務レビューで確認要 |
| **Tier 1（AI候補のみ、未レビュー）** | 1年（レビューされなければ自動削除） | 即時削除可能 | 利用目的の達成（第19条） |
| **端末側のraw位置ログ** | 端末上に90日。サーバーには送信しない | ユーザーが端末上で随時削除可能 | N/A（サーバー上にないため） |
| **端末側のraw音声/画像** | 端末上に30日。サーバーには送信しない | ユーザーが端末上で随時削除可能 | N/A |
| **候補イベントメタデータ（サーバー上）** | 3年 | 削除要求から30日以内に処理 | 同意ベース |
| **ユーザーアカウント情報** | アカウント存続中＋退会後1年 | 退会時に自動削除。紐づくデータは匿名化 | 同意ベース |

## A4. 研究者アクセス制御

### A4.1 アクセスTier

| accessTier | 対象 | アクセスできるデータ | 条件 |
|------------|------|-------------------|------|
| **public** | 誰でも | coordinatePrecisionで丸められた座標、Tier 2以上の同定結果、evidence thumbnail | なし |
| **researcher** | 登録研究者 | 希少種の精密座標、高解像度evidence | 研究者登録＋利用目的申請＋所属機関確認 |
| **admin** | 運営者 | 全データ（匿名化前含む） | 運営権限 |

### A4.2 研究者アクセスの手続き

1. 研究者がikimon.lifeに登録申請（氏名、所属、研究テーマ、利用目的）
2. 運営が所属機関を確認（大学/博物館/研究機関のメールドメイン等）
3. 利用目的の審査（商用利用は不可。学術研究・保全活動に限定）
4. アクセス承認。有効期間1年（更新制）
5. アクセスログは全件記録。四半期ごとに監査

### A4.3 DSAR（Data Subject Access Request）SLA

| 要求種別 | 対応期限 | 備考 |
|---------|---------|------|
| 開示請求 | 受理から30日以内 | APPI第33条準拠 |
| 訂正請求 | 受理から30日以内 | APPI第34条準拠 |
| 削除請求（Tier 1） | 受理から14日以内 | 即時削除可能なデータ |
| 削除請求（Tier 2以上） | 受理から30日以内 | 匿名化で対応。完全削除が必要な場合は個別判断 |

## A5. 国外移転

| 移転先 | データ種別 | 条件 |
|--------|-----------|------|
| **GBIF（コペンハーゲン）** | DwC-A exportデータ（座標丸め済み、recordedBy匿名ID化済み） | **非個人データへの加工**（A2と同じ整理）。座標丸め＋匿名ID化により個人データに該当しないレベルまで加工した上でexport。非個人データの提供であるため、APPI第28条（外国にある第三者への提供制限）の適用対象外。※「匿名加工情報」（第2条第6項）としての整理ではない |
| **Google（Gemini API）** | テキストメタデータ（候補イベントの記述文のみ） | raw個人情報は含まない。Googleのデータ処理契約に基づく |
| **その他** | なし | 新規移転先が発生した場合はこのAppendixを更新 |

## A6. 監査ログ項目

| 項目 | 記録内容 |
|------|---------|
| アクセスログ | who（ユーザーID/研究者ID）、what（アクセスしたレコードID）、when（タイムスタンプ）、accessTier |
| 変更ログ | レコードの作成・更新・削除。変更前後の値。変更者。変更理由 |
| 同定ログ | Identification Layerの全操作。auditLogフィールドに記録 |
| 削除要求ログ | 要求者、対象レコード、要求日時、処理結果、処理日時 |
| export ログ | DwC-A exportの実行日時、対象レコード数、export先 |
