## C3. 数値ゲート（Success / Fail Gate）

### C3.1 Go/No-Go 判定基準

| ゲート | 指標 | 成功条件 | 失敗条件 | 測定方法 |
|--------|------|---------|---------|---------|
| **G1: 検出量** | 1時間あたりTier 1イベント数 | ≥ 10件/時間 | < 3件/時間 | BirdNET出力のイベントカウント |
| **G2: AI精度** | Tier 1.5自動昇格の正答率（reviewer検証ベース） | ≥ 80% | < 60% | Tier 1.5のうちreviewerがTier 2に承認した割合 |
| **G3: レビュー速度** | 1件あたりのreviewer工数 | ≤ 3分/record（中央値） | > 10分/record | reviewer操作ログ |
| **G4: 昇格速度** | Tier 1→Tier 2 所要時間 | ≤ 72時間（中央値） | > 7日 | タイムスタンプ差分 |
| **G5: reviewer持続性** | Active reviewer数（月末時点） | ≥ 5名 | < 3名 | 月に1件以上レビューした人数 |
| **G6: サーバー負荷** | VPS CPU使用率（ピーク） | < 70% | > 90% | サーバーモニタリング |
| **G7: ストレージ** | 3ヶ月間の総ストレージ消費 | < 50GB | > 200GB | df -h |

### C3.2 判定ルール

- **Go（Phase 6本格開発に進む）：** G1-G5が全て成功条件を満たす
- **Conditional Go（設計修正して継続）：** G1-G5のうち1-2件が失敗条件に近い。原因を特定し設計修正
- **No-Go（アプローチ再検討）：** G1-G5のうち3件以上が失敗条件

G6-G7はインフラ制約であり、Go判定とは独立。失敗条件に達した場合はインフラ拡張で対応。

## C5. 顧客PoC（愛管株式会社向け）— 第2段階

**重要：音響PoCが主、顧客PoCは観察/ヒアリング層。** 2つのプロジェクトを同時に走らせるのではなく、Phase B-Cの音響実証の結果を見ながら、愛管株式会社への観察・ヒアリングを並行で進める。Go判定はG1-G5の音響PoCで行い、顧客PoCは「使えそうか？」の定性検証。

### C5.1 目的

OECM/TNFDモニタリング基盤としてのikimon.lifeの価値を、1社で定性的に検証する。本格的な顧客PoC（有料サービスとしての検証）はG1-G5がGoの後。

### C5.2 意思決定者

| 役割 | 対象 | 確認事項 |
|------|------|---------|
| 意思決定者 | 愛管株式会社の自然共生サイト責任者（施設管理 / 総務 / サステナビリティ担当のいずれか。初回ヒアリングで氏名・役職を確定） | 「このデータがあれば自然共生サイトの申請/維持が楽になるか？」 |
| 技術協力者 | 八巻（ikimon.life開発者） | パッシブセンシング＋Trimble X7でのデータ収集 |
| 評価者 | 外部の生態学者（可能であれば） | Tier 3データの科学的妥当性評価 |

### C5.3 成果物

| 成果物 | 内容 | 期限 |
|--------|------|------|
| 候補地の音響種リスト | BirdNETで検出した鳥類リスト（Tier 1.5以上） | 5月末 |
| Tier 3到達サンプル | 複数reviewer合意に至った種の同定記録（5件以上） | 6月中旬 |

| 顧客フィードバック | 担当者名・役職、5段階評価、「使えそうか？足りないものは何か？」、導入条件、次アクションを含むヒアリング記録 | 6月末 |

### C5.4 ICP（Ideal Customer Profile）仮説

| 項目 | 仮説 |
|------|------|
| 業種 | 製造業、建設業、不動産業（広い自社敷地を持つ企業） |
| 規模 | 従業員100名以上（CSR/サステナビリティ部門がある） |
| ニーズ | 自然共生サイト（OECM）認定の取得・維持、TNFD開示への対応 |
| 予算 | 月額5-30万円（現行の専門調査費用の1/10以下が目安） |

→ 愛管株式会社での実証でこの仮説を検証する。**意思決定者の氏名・役職が記録されていない場合、顧客PoCは完了扱いにしない。**

## C6. リスクと対策

| リスク | 影響 | 対策 |
|--------|------|------|
| BirdNETの精度が候補地で低い | G2失敗 | 種リストのローカライズ（日本の鳥類に特化したfine-tuning検討）。精度が低い種はTier 1.5から除外 |
| reviewerが集まらない | G5失敗 | 八巻のネットワークで最低5名確保。足りなければ大学の分類学研究室にアプローチ |
| Xserver VPSの負荷超過 | G6失敗 | event-only ingest設計で軽量化済み。超過した場合はプラン変更（12GB→24GB） |
| 愛管株式会社の協力が得られない | 顧客PoCが中止 | 代替サイト（公園、河川敷等の公共地）で音響PoCだけ先行 |
| 季節的にデータが薄い（4-6月は鳥の繁殖期で良い時期） | N/A | むしろベストシーズン。問題なし |
