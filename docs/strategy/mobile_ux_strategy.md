# ikimon.life モバイルアプリ UX戦略

**作成日**: 2026-03-19
**対象**: Phase 6 パッシブ観察プラットフォーム
**プラットフォーム**: Android (Pixel 10 Pro) 先行 → iOS 後追い
**原則**: ユーザー体験優先。技術は裏方。**主役は市民。**

---

## 市民ファースト原則（最上位ルール）

1. **「今日の散歩で何がいたか」が最短で見える** — これが全ての出発点
2. **「名前がわかる」喜びが持続する** — AI の断定精度は過大表現しないが、「〜かもしれません」の提案は積極的に出す
3. **「蓄積が資産になる」実感** — 1週間・1ヶ月・1年の振り返りで「こんなに見てたんだ」
4. **3D はご褒美、日常は2D** — 3Dモデルはエリアページの「すごい体験」として提供。毎日のフィードは従来のカード形式
5. **Eagle は「プロモード」** — 一般ユーザーは知らなくていい。使える人が使うと、みんなのエリアがリッチになる
6. **技術をひけらかさない** — 「SLAM」「COPC」「TimescaleDB」はユーザーの目に触れない。ユーザーが見るのは「あ、シジュウカラだ！」

---

## コンセプト: 「自然は、見えている」

普通の人は自然を「見ていない」のではなく「気づいていない」。
ikimon.life のモバイルアプリは、**気づきのレイヤー**を日常に重ねる。

## 2つのモード

### モード1: ポケット（Passive）

**一言**: 「歩くだけで、自然が記録される」

```
ユーザーの行動:
  1. アプリを開いて「ポケットモード ON」
  2. スマホをポケットに入れる
  3. いつも通り散歩する
  4. 帰宅後、通知が来る:
     「今日の散歩: 推定6種の鳥声を検出しました 🐦」

必要な操作: ゼロ
```

**UX の鍵**:
- **ゼロ操作**: ボタン1つでON、あとは触らない
- **驚きの発見**: 「いつもの公園にこんなにいたの？」
- **蓄積の可視化**: 毎日歩くだけで「マイ生態系マップ」が育つ
- **省電力**: 1時間で5%以下のバッテリー消費

```
  │   → 信頼度の総合判定
  └── DataStageManager → ai_classified ステージ設定
```

### モード2: スキャン（Active Scan）

**一言**: 「カメラを通して、自然が名前を持つ」

```
ユーザーの行動:
  1. 「スキャンモード」をタップ
  2. カメラを風景に向ける
  3. 画面上に種名がリアルタイムで浮かぶ
  4. 歩き回るだけで、見えた生物が自動記録される

必要な操作: カメラを向けるだけ
```

**UX の鍵**:
- **ゲーム感覚**: ポケモンGO的な「見つけた！」の快感
- **シャッター不要**: 信頼度が高い検出は自動キャプチャ
- **リアルタイム**: カメラ映像にラベルが浮かぶ AR 体験
- **累計カウンター**: 「今日のスキャンで12種発見！」

```

  │
  ├── 2段階推論パイプライン:
  │   Stage 1: YOLO nano (物体検出)

  │   → 同じ個体の連続検出を1件にまとめる

  ├── 信頼度高 → 自動投稿 (ai_classified)
  ├── 信頼度中 → ユーザー確認待ち
  └── 信頼度低 → 同定キューに流す
```

## UX原則

### 1. 操作はゼロに近づける

| レベル | 操作 | モード |
|---|---|---|
| **0操作** | 歩くだけ | ポケット |
| **1操作** | カメラを向ける | スキャン |
| **2操作** | シャッターを押す | 従来の投稿 |

下に行くほど「意識的」。上に行くほど「自然」。
ikimon.life は上を目指す。

### 2. 驚きファースト

「やらなきゃ」ではなく「え、こんなにいたの！？」。
- サマリー通知は **発見の驚き** を伝える
- 数字より「シジュウカラが3回も来てた」という物語
- 先月との比較で「変化」を見せる

### 3. 蓄積が資産になる

- 毎日の散歩 → 「マイ生態系マップ」が育つ
- 1年続けると → 季節変動が見える
- 10年続けると → 環境変化の証人になる
- **100年分のデータ** → 人類の宝

### 4. プライバシー・バイ・デザイン

- 音声は推論後に **即破棄**（録音は保存しない）
- 位置情報は PrivacyFilter のグリッド丸め適用
- 希少種は自動マスク
- ユーザーのルートは本人だけが見える

## データフロー（サーバー側）

```
端末 → api/v2/passive_event.php
          ↓
   PassiveObservationEngine.php
     ├── 位置×時間×季節 → 既知種候補との照合
     ├── オンデバイス検出結果との突合
     ├── 信頼度の総合判定
     └── DataStageManager.php
           ├── 高信頼度 → ai_classified (自動記録)
           ├── 中信頼度 → needs_review (ユーザー確認待ち)
           └── 低信頼度 → unverified (コミュニティ同定待ち)

端末 → api/v2/scan_detection.php
          ↓
   同上のフロー + 写真データ処理
```
