# ウォーク・ライブスキャン・たんけんマップ統合計画

> **「2026年に記録しなかったデータは二度と取り戻せない」**

## 問題: 3つの体験がバラバラ

現在、フィールド体験が3ページに分散している:

| ページ | 機能 | 問題 |
|--------|------|------|
| `field_research.php` | GPS追跡 + Fog of War + SiteGuide | スキャン機能がない。歩くだけ |
| `field_scan.php` | カメラ + 音声 + GPS リアルタイム検出 | Fog of Warがない。たんけんマップと断絶 |
| `walk.php` | 音声検出のみ | カメラがない。field_scan の下位互換 |

**ユーザーの混乱**: どのページを使えばいい？ 3つの差がわかりにくい。
**データの断絶**: field_research のGPS軌跡と field_scan の検出データが別セッション。
**哲学に反する**: 複雑さは参加率を落とし、記録総量を減らす。

---

## 解決: 1つに統合する

### 「さんぽ」= field_research.php に全部載せる

`field_research.php` を**唯一のフィールド体験ページ**にする。

```
統合後の field_research.php:
┌─────────────────────────────────────────┐
│  🗺️ たんけんマップ（Fog of War）       │ ← 既存
│  📍 GPS軌跡リアルタイム表示             │ ← 既存
│  📖 SiteGuide パネル                   │ ← 既存
│                                         │
│  + 📷 ライブスキャン（カメラAI検出）    │ ← field_scan.php から統合
│  + 🎤 音声検出（鳥音声AI）              │ ← walk.php から統合
│  + 🐦 検出カード（種名+信頼度）         │ ← field_scan.php から統合
│  + 📊 セッション終了レポート            │ ← field_scan.php から統合
│                                         │
│  ボタン: [さんぽ開始] → [終了]         │
└─────────────────────────────────────────┘
```

### walk.php と field_scan.php は非推奨化

- `walk.php` → `field_research.php` にリダイレクト
- `field_scan.php` → `field_research.php` にリダイレクト
- 旧URLは1ヶ月間リダイレクトを維持

## 統合後のユーザーフロー

### 1. 開始画面

```
┌─────────────────────────────────────────┐

│  🚶 あるく: GPS + 音声検出（ポケットOK）│
│  📷 スキャン: 上記 + カメラAI検出       │
│  🔇 静か: GPSのみ（バッテリー節約）    │
│                                         │
│  [さんぽ開始]                           │

```

**3モードの考え方:**
- **あるく**: ポケットに入れて歩く。音声AIが鳥の声を拾う。GPS軌跡 + Fog of War。**これがデフォルト。**
- **スキャン**: カメラを構えて歩く。あるくモードの全機能 + 視覚AI。field_scan.php の後継。
- **静か**: GPSだけ。たんけんマップを埋めたいがバッテリーを節約したい時。

### 4. セッション終了レポート（全面リデザイン）

**現状の問題**: field_scan.php の `#scan-done` は情報が多すぎて散漫。

**新設計: 「散歩レポート」= 自然版 Fitbit レポート**

```
┌─────────────────────────────────────────┐
│  🌿 今日のさんぽ                       │
│  浜松城公園 · 3月24日                   │
│                                         │
│  ┌─────────────────────────────────┐   │
│  │  🚶 45分 · 1.2km · 4,800歩     │   │
│  │  🐦 12種に出会えた              │   │
│  │  🌿 自然浴スコア: 7.8/10        │   │
│  └─────────────────────────────────┘   │
│                                         │
│  ── 出会った生きもの ──                 │
│  ┌────┐ ┌────┐ ┌────┐ ┌────┐        │
│  │🐦  │ │🐦  │ │🐦  │ │🦋  │        │
│  │ｼｼﾞｭ│ │ﾒｼﾞﾛ│ │ｳｸﾞｲ│ │ｱｹﾞﾊ│        │
│  │ 92% │ │ 87% │ │ 76% │ │ 84% │        │
│  └────┘ └────┘ └────┘ └────┘        │
│  （横スクロール、タップで詳細）         │
│                                         │
│  ── AIストーリー ──                     │
│  "春の林縁を歩いた45分。シジュウカラの │
│   活発なさえずりが印象的でした。        │
│   ウグイスの初鳴きも確認..."           │
│                                         │
│  ── あなたの貢献 ──                     │
│  🗺️ この場所は3回目のスキャンです       │
│  📈 前回(3/17)より2種多く記録          │
│  🌡️ 気温データが気候モニタリングに貢献  │
│                                         │
│  ── 今週の記録 ──                       │
│  📊 3回 · 27種 · 3.6km                │
│  🔥 5日連続さんぽ中!                   │
│                                         │
│  [🗺️ たんけんマップを見る]             │
│  [📸 ベスト写真を投稿]                 │
│  [🔄 もう一回さんぽ]                   │
└─────────────────────────────────────────┘
```

**設計原則:**
1. **最初に見えるのは「達成感」**: 時間・距離・種数・自然浴スコア
2. **種カードは横スクロール**: 縦長にしない。サクッと眺められる
3. **AIストーリーは短く**: 3行以内。読み飛ばせる
4. **貢献メッセージは文脈依存**: session_recap.php の既存9パターンを活用
5. **週間累計を見せる**: 継続の実感。Fitbit の週間サマリーと同じ
6. **たんけんマップへの導線**: 「Fog of War がどれだけ開いたか」を見に行ける

### 自然浴スコアの算出

```
自然浴スコア (0-10):
  = (種の多様性 × 0.4) + (音風景の質 × 0.3) + (環境の自然度 × 0.3)

種の多様性:
  - 検出種数を期待種数（Chao1推定 or エリア平均）で割る
  - 12種/推定18種 = 67% → 6.7点

音風景の質（FieldScan のみ、Web版は推定値）:
  - NDSI（自然音/人工音比）を0-10にマッピング
  - NDSI 0.6 → 7点

環境の自然度:
  - Nano 判定 or エリア属性（森林/公園/市街地）
  - 森林 → 9点、公園 → 7点、市街地 → 4点
```

Web版（Live Scan）でもセンサーなしで推定値を出せる。FieldScan があればセンサーで正確な値。

## 技術的な統合方針

### field_research.php の拡張

```
現在の field_research.php:
  Alpine.js explorationApp
  ├── FieldRecorder.js (GPS)
  ├── ExplorationMap.js (Fog of War)
  └── SiteGuide.js (エリアガイド)

統合後:
  Alpine.js explorationApp
  ├── FieldRecorder.js (GPS) ← 既存
  ├── ExplorationMap.js (Fog of War) ← 既存
  ├── SiteGuide.js (エリアガイド) ← 既存
  ├── LiveScanner.js (カメラ+音声検出) ← field_scan.php から移植
  ├── SessionReporter.js (終了レポート) ← 新規
  └── NatureScore.js (自然浴スコア計算) ← 新規
```

### データフロー統一

```
field_research.php (統合版)
  ├── GPS: FieldRecorder → sessionStorage
  ├── 視覚: LiveScanner → passive_event.php
  ├── 音声: LiveScanner → passive_event.php
  └── 終了: SessionReporter → session_recap.php + scan_summary.php

passive_event.php は変更不要（既に全データタイプ対応済み）
session_recap.php も変更不要（既に AI要約 + 貢献メッセージ対応済み）
```

### 新規: NatureScore API

```
POST /api/v2/nature_score.php
{
  "species_count": 12,
  "duration_sec": 2700,
  "distance_m": 1200,
  "area_type": "park",         // GPS → GeoContext で自動判定
  "acoustic_ndsi": 0.6,        // FieldScan のみ（Web は null）
  "detections": [...]
}

→ Response:
{
  "score": 7.8,
  "breakdown": {
    "diversity": 6.7,
    "soundscape": 7.0,  // null なら area_type から推定
    "environment": 7.0
  },
  "message": "豊かな春の林を楽しめたさんぽでした"
}
```

## FieldScan（Android）との関係

FieldScan は**この統合版 field_research.php の「ネイティブ版」**。

| 機能 | Web (field_research.php) | FieldScan (Android) |
|------|--------------------------|-------------------|
| GPS追跡 | ✅ | ✅ |
| Fog of War | ✅（ブラウザ） | ❌（アプリ内マップは後回し） |
| 視覚AI | ✅ Gemini Flash API | ✅ TFLite オンデバイス |
| 音声AI | ✅ サーバーサイドAPI（将来Perch v2移行） | ✅ Perch v2 オンデバイス |
| 環境センサー | ❌（ブラウザの限界） | ✅ 温度/気圧/照度/音響指数 |
| オフライン | ❌ | ✅ |
| 終了レポート | ✅ session_recap.php | ✅ 同じAPI |
| 自然浴スコア | ✅（推定値） | ✅（センサー実測値） |
| データ送信先 | passive_event.php | passive_event.php（**同じ**） |

**FieldScan のセッション結果もブラウザで見る。** アプリ内に結果ページを作り込む必要はない。
FieldScan セッション終了 → ikimon.life の結果ページURLを開く → 統合された散歩レポートを表示。
