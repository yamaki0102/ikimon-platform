# ADR-002: 100年保全戦略 (Century Durability)

**ステータス**: 承認
**日付**: 2026-03-19
**影響範囲**: 全データ設計・保存・エクスポート

## 背景

ikimon.life は「100年分の生態系変遷を記録する」プラットフォーム。2126年の研究者がデータを再利用できるためには、特定ベンダー・モデル・座標系・分類体系に依存しない設計が必要。

## 決定

### 100年後テスト

全ての設計判断は以下の5項目を通過すること:

1. このデータは、**特定ベンダーや特定アプリが消えても**読めるか
2. この観測は、**今のAIモデル名が無意味になっても**再評価できるか
3. この位置は、**基盤地図や座標変換手法が変わっても**再投影できるか
4. この分類結果は、**分類体系が更新されても** taxon concept を追跡できるか
5. このシステムは、**開発者が全員入れ替わっても**引き継げるか

1つでも No なら、その設計は再検討すること。

### データ永続化ルール

| ルール | 説明 |
|---|---|
| raw は immutable | 一度保存した観測イベントは変更しない |
| derived は再生成可能 | raw から再計算できる（organisms, scores, heatmaps） |
| version stamp 必須 | schema_version, pipeline_version, model_version, taxonomy_version |
| checksum 付与 | 各アセットに SHA-256 ハッシュ |
| provenance chain | 各処理の来歴を記録（誰が、いつ、何を、なぜ） |
| export で再構築可能 | export パッケージだけでシステムを復元できる最小メタデータ |
| ベンダー閉じ込め | 外部 API は adapter 層に隔離。正本は ikimon 側で保持 |

### ファイルフォーマット方針

| データ型 | 正本フォーマット | 理由 |
|---|---|---|
| 観測イベント | PostgreSQL + JSON export | リレーショナル + 可搬性 |
| 点群 | COPC / LAZ | OGC 標準、100年読み取り保証 |
| 3D 配信 | 3D Tiles | OGC 標準 |
| 写真 | JPEG / WebP + EXIF | 広く普及 |
| メタデータ | JSON / CSV | 最も単純 |
| GIS エクスポート | GeoPackage | OGC 標準、SQLite ベース |

### 禁止事項

- vendor 固有のバイナリフォーマットだけに依存して長期保存しない
- closed SaaS を唯一の正本保管先にしない
- schema migration を口伝で済ませない（ADR で記録する）
- 「いま動く」だけで archive/export を後回しにしない

### 定期アーカイブ

- 月次: 観測データの DwC-A エクスポート
- 四半期: 全データの GeoPackage エクスポート
- 年次: 完全バックアップ + メタデータ manifest
