## 1. 国際フレームワーク動向

### 1.1 TNFD LEAP (Taskforce on Nature-related Financial Disclosures)

**概要**: 自然関連財務情報開示のための国際フレームワーク。4フェーズで構成。

| フェーズ | 内容 | ikimonの対応可能性 |
|:---|:---|:---|
| **L**ocate | 自然との接点の特定（地理、セクター、バリューチェーン） | ✅ サイト登録＋GeoJSON境界で自動対応 |
| **E**valuate | 自然への依存度・影響の評価 | ✅ BISスコア＋レッドリスト照合で対応済 |
| **A**ssess | 自然関連リスク・機会の分析 | 🔶 経年トレンド分析の強化が必要 |
| **P**repare | 開示情報の準備・報告 | ✅ PDFレポート生成で対応済 |

**出典**: [tnfd.global](https://tnfd.global), Linklaters TNFD Guide 2025, Mitsui TNFD Implementation Report

**重要な動向**:
- TNFDは2025年時点でAI技術の活用を推進。特にSME（中小企業）向けのアクセシブルなツール開発を奨励
- 複数データソース（衛星、ドローン、地上、固有データ）の「証拠の収斂」が推奨されている
- 気候と自然の統合シナリオ分析が次のフロンティア

### 1.2 CSRD / ESRS E4 (EU企業サステナビリティ報告指令)

**概要**: EU域内の大企業に義務付けられるサステナビリティ報告基準。ESRS E4が生物多様性・生態系を規定。

**主要要件**:
- **ダブルマテリアリティ**: 事業→生物多様性 と 生物多様性→事業 の双方向評価
- **125データポイント**: うち60が必須（2025年11月のEFRAG改訂で61%削減予定）
- **ロケーションベース報告**: 影響が発生する場所の特定、生物多様性センシティブエリアの有無
- **適用開始**: 2024年度分から段階適用。大企業は2025年度分（2026年公開）

**出典**: EFRAG ESRS E4 Final Draft, DarwinData.ai Analysis, Coolset.com ESRS Guide

**ikimonへの示唆**:
- ESRS E4の構造化データエクスポート（JSON）を実装すれば、EU企業のレポーティングツールに直接取り込み可能
- 「ロケーションベース」はikimonのサイト単位管理と完全に合致

### 1.3 SBTN (Science Based Targets for Nature)

**概要**: 科学的根拠に基づく自然関連目標設定フレームワーク。

**企業アセスメントプロセス**:
1. **Step 1: Assess** — Materiality Screening Tool (MST) で重要テーマを自動特定
2. **Step 2: Prioritize** — 地理的・圧力カテゴリ別の優先順位付け
3. **Step 3: Measure, Set & Disclose** — 目標設定と開示

**自動化の現状**:
- MSTは「一貫した自動結果」を提供する設計
- 上流活動の自動リストアップ（セクター間経済投入産出関係を利用）
- Self-Assessment Toolが公式提供されている

**出典**: sciencebasedtargetsnetwork.org Corporate Manual, Accountability Accelerator

### 1.4 IPBES 2026 ビジネスと生物多様性評価

**概要**: 2026年2月承認の最新国際評価。ビジネス・政府・金融に100超の具体的行動を提言。

**核心メッセージ**:
- 生物多様性喪失はシステミックリスク（個別企業ではなく経済全体のリスク）
- 自発的行動だけでは不十分 → 政策・金融システムの整合が必要
- ビジネスが依存する生態系サービス（水供給、花粉媒介、気候調節）の劣化が財務リスクに直結

**出典**: unep-wcmc.org, businessfornature.org, europa.eu

## 4. 市民科学の企業活用エビデンス

### 4.1 iNaturalistの企業活用事例

**メリット**:
- 広範な地理的カバレッジ、大規模データ収集が低コストで可能
- ベースライン確立、復元効果モニタリングに有効
- 従業員・地域住民のエンゲージメント向上
- Collection Projectsで地理的・グループ別に制限可能

**限界**（ikimonの差別化ポイント）:
- iNaturalistは非営利プラットフォームであり、企業向けSaaSではない
- TNFDやCSRDへの直接的なレポート出力機能がない
- 企業固有のサイト管理（GeoJSON境界、ポートフォリオ管理）がない
- SLA（サービスレベル保証）やサポートがない

**出典**: wikipedia.org (iNaturalist), undp.org, sustainability-directory.com, inaturalist.org

### 4.2 市民科学×企業の学術的根拠

- 市民科学は従来の生態モニタリングの「範囲」を拡大
- 大規模・長期間のデータ収集が可能
- 費用対効果が高い（専門家のみに依存しない）
- CSR/SDGs報告の「ストーリー」として活用可能

## 5. 技術トレンド

### 5.1 BISスコアの科学的根拠

**Shannon-Wiener Index (H')**:
- 世界で最も広く使われる生物多様性指標の一つ
- `H' = -Σ(pi × ln(pi))` （pi = 各種の個体数割合）
- 種の豊富さ（richness）と均等度（evenness）を同時に評価
- 通常1.5〜3.5の範囲、これを0-100に正規化可能

**IUCNカテゴリ加重**:
- CR（絶滅危惧IA類）= 5倍
- EN（絶滅危惧IB類）= 4倍
- VU（絶滅危惧II類）= 3倍
- NT（準絶滅危惧）= 2倍
- LC（軽度懸念）= 1倍

**利点**: 国際共通、国ごとの制度差に依存しない

### 5.2 eDNA vs 市民参加型モニタリング

| 項目 | eDNA (NatureMetrics) | 市民参加型 (ikimon) |
|:---|:---|:---|
| 検出能力 | 数千種同時（高精度） | 目視可能種のみ |
| コスト | 1サイト数十万円〜 | 基本無料（参加者の時間のみ） |
| リードタイム | 数週間（ラボ解析） | リアルタイム |
| エンゲージメント | なし | 高い（地域連携） |
| スケーラビリティ | ラボキャパシティに依存 | 参加者数に比例 |
| TNFD対応 | 部分的 | 全フェーズ対応可能 |

**結論**: 両者は**補完的**であり、競合ではない。ikimonは「日常的モニタリング」、eDNAは「ベースライン精密調査」として共存可能。
