# SZ-UC-05 牧之原の茶農家が益虫害虫を記録する

## 1. 概要

- **目的**: 実用目的の利用者、とくに高齢寄りユーザーが、農地で見つけた虫を無理なく記録できるかを検証する
- **主対象**: 高齢者 UI、実用性、文字量、片手操作、写真中心入力
- **優先度**: 高
- **想定所要時間**: 20 分

## 2. ユーザー像

- 牧之原市の茶農家、60 代
- スマホは日常利用するが、細かい UI は苦手
- 生き物好きというより、畑管理の一環で知りたい

## 3. シナリオ

5 月、茶畑でテントウムシ、クモ、蛾の幼虫、ハチ類を見かけた。  
それが益虫なのか害虫なのか判断材料を得るため、まず写真付きで記録したい。

## 4. 前提条件

- モバイル表示で検証する
- 屋外の片手操作を想定する
- 写真 1 枚を用意する
- 種名は分からない前提で進める

## 5. 必要な入力データ

- 仮ラベル: `茶の木にいた幼虫`
- メモ例: `新芽の近くに複数いた`
- 日時: 5 月の午前
- 位置: 牧之原市の茶畑

## 6. 検証タスク

- タスク A: 写真中心で最小限の入力を行う
- タスク B: 種名不明でも記録を完了する
- タスク C: 農業文脈で使う価値があるか判断する

## 7. 操作手順

1. 片手操作を意識して投稿を開始する
2. 写真を追加する
3. 文字入力を最小限にして進める
4. 場所と日時を設定する
5. 投稿する
6. 投稿後に見返しやすさを確認する

## 8. 観察ポイント

### 操作負荷

- タップ対象が十分大きいか
- 文字入力なしでもある程度進められるか
- 片手操作で厳しい箇所がないか

### 実用性

- 自然観察アプリに見えすぎて農業用途が浮いていないか
- 見返した時に「畑管理メモ」として使えるか

### 種名不明時の UX

- 分からないことを責めない設計になっているか
- 写真さえあれば記録できる印象があるか

## 9. 合格条件

- 5 分以内に投稿できる
- 片手操作で致命的な詰まりがない
- 文字入力が少なくても記録価値がある
- 農家ユーザーが「また畑で使える」と感じる

## 10. 失敗シグナル

- 小さな UI や専門用語が多く高齢者に厳しい
- 実用文脈より趣味アプリ感が強すぎる
- 種名不明のままでは保存しにくい
- 投稿後に畑の記録として見返しづらい

## 11. 深掘り質問

- 屋外の短時間利用に耐えたか
- 片手で使いづらい箇所はどこか
- 農作業の流れに組み込めそうか
- 専門知識がなくても意味のある記録になったか

- 問題点:
- 現場利用適性（1-5）:
- 総合判定: 合格 / 要改善 / 不合格
