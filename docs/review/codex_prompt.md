# Codex 投入プロンプト

以下をそのままコピーして Codex に貼り付けてください。

---

あなたは、ソフトウェアアーキテクチャ・リモートセンシング・市民科学プラットフォーム・3D空間情報・保全生態学の全てに精通した、世界最高水準の技術アドバイザーです。

私は ikimon.life という市民参加型生物多様性プラットフォームを開発しています。以下の3つのデバイスを組み合わせて、エリアの生態系3Dデジタルツインを構築するシステムを設計しました:

- **3DMakerPro Eagle**: ハンドヘルド LiDAR スキャナー（200,000pts/sec、140m範囲、2cm精度、8K HDR パノラマ、3D Gaussian Splatting出力）
- **iPhone 13 Pro**: カメラ種検出（Core ML）+ 近距離LiDAR + ARKit
- **Pixel 10 Pro**: バックグラウンド音声検出（BirdNET/Gemini Nano）+ GPS常時追跡

コンセプトは「Eagleでエリアの3D空間基盤を構築し、スマホで毎日の散歩中に生物データを蓄積し、100年分の生態系変遷を3D空間として記録する」というものです。

このリポジトリの `docs/review/codex_review_directive.md` に、設計の全体像・ハードウェアスペック・現在のアーキテクチャ・11セクションのレビュー項目をまとめた指示書があります。

**まず `docs/review/codex_review_directive.md` を全文読んでください。**

その上で、指示書に記載された全11セクション（4.1〜4.11）の全ての質問に、1つも飛ばさず回答してください。

回答にあたっての絶対ルール:

1. **お世辞ゼロ**。「よくできている」「素晴らしい」は禁止。問題点・見落とし・もっと良い方法だけを出力せよ
2. **全ての指摘に具体的な代替案を添えよ**。「〜が問題」だけでは不十分。「〜が問題。代わりに〜すべき。理由:〜」の形式で
3. **妥協案は不要**。予算・人員・時間の制約は考慮しなくていい。技術的に最善の選択肢だけを提示せよ
4. **業界の先行事例・ベンチマーク・論文があれば引用せよ**
5. **回答は日本語で**
6. **4.11 の最終判定（7問）には必ず明確に回答せよ**。特に「このプロジェクトが失敗するとしたら最も可能性の高い原因は何か」と「iNaturalist を超えるために最も重要な1つの決断は何か」は、曖昧な回答を禁止する

補足情報:
- 現在のユーザー向け価値は「散歩中に生物の名前がわかる + コミュニティで同定し合える」。3Dモデルは長期的な蓄積価値（30年後に振り返れる生態系の記録）だが、今日のユーザーの日常的な喜びとは直結しない可能性がある。この点も踏まえて UX を評価してほしい
- バックエンドは PHP 8.2 の共有サーバー。VPS 移行は Phase 4（6ヶ月後）に予定。現時点では共有サーバーの制約内で動く必要がある
- テストは128本、全パス。コードベースは PHP 81ファイル + API 74エンドポイント + iOS Swift + Android Kotlin

では、`docs/review/codex_review_directive.md` を読んだ上で、全項目に回答してください。
