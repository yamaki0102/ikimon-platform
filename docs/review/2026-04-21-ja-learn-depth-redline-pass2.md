# 2026-04-21 JA Learn Depth Redline Pass 2

staging 日本語コピー第3段の赤入れメモ。対象は `learn-field-loop`, `learn-authority-policy`, `learn-methodology` の 3 ページに限定する。

## learn-field-loop

### 現状の弱点

- 循環の考え方は見えるが、`なぜ循環で考えるのか` が冒頭でまだ弱い。
- `1 件で言い切らない理由` と `再訪で何が増えるか` は本文に散っていて、初心者が拾いづらい。
- AI の役割は書いてあるが、`どこで補助し、何は決めないのか` の線引きがもう一段ほしい。

### 残すもの

- `見つける -> 記録する -> また歩く` の軸
- `同じ場所に戻るほど見えるものが増える` という考え方
- `AI は観察の代わりではない`

### 追加すべき説明

- なぜ 1 件を終点ではなく途中経過として扱うのか
- 再訪で `差分` だけでなく `見える範囲` そのものが増えること
- 読んだあとに、実際の散歩や記録でどう使えばよいか

### 書き換え方針

- 冒頭で `なぜ循環で考えるのか` を正面から答える。
- `なぜ 1 件で言い切らないのか`, `再訪で何が増えるのか`, `AI はどこで補助するのか` を独立見出しにする。
- 最後に `次にどう使うか` を入れて、考え方のページで終わらせない。

### 反映後の確認観点

- lead を読むだけで `また歩く / 1 件で言い切らない / AI は補助` が取れる。
- 本文で `理由 / 線引き / 次の使い方` が順に回収される。

## learn-authority-policy

### 現状の弱点

- 段階を分ける理由はあるが、`ふだん使う人は何を見ればよいか` がまだ暗黙になりやすい。
- `AI / みんなの見立て / 任された人 / 公開前判断` の違いはあるが、どこで慎重さが必要になるかが少し抽象的。
- 制度の説明としては良いが、初心者の不安を減らす導線が弱い。

### 残すもの

- 同定を 1 段で済ませない設計
- `名前が付いた` と `外に出してよい` を分ける考え方
- 推薦と監査を残す方針

### 追加すべき説明

- なぜ段階を分けるのか
- どんなときに慎重さが必要になるのか
- 普段の利用で読み手がまず見るべきポイント

### 書き換え方針

- lead に `AI の候補 / みんなの見立て / 任された人の確認 / 公開前判断` を残しつつ、`どこで慎重さが必要か` を明記する。
- 本文に `ふだん使うときはどう読めばよいか` を追加し、制度説明だけで終わらせない。
- `慎重さが必要な場面` を独立見出しにして、初心者にも線引きが分かるようにする。

### 反映後の確認観点

- 1 画面目で `段階を分ける理由` と `慎重さが必要な場面` が読める。
- 本文で `段階の違い / 読み方 / 責任の所在` が追える。

## learn-methodology

### 現状の弱点

- 場所と公開範囲の説明はあるが、`なぜ場所を残すのか` が冒頭でまだ軽い。
- `見ていないことを言い切らない理由` はよいが、普段使いで何を気にすればよいかが薄い。
- ルールの説明としては成立しているが、読み手が `じゃあ自分は何を守ればよいか` に落とし込みづらい。

### 残すもの

- 場所を残すことと公開範囲を分ける考え方
- `まだ見ていない` と `いない` を分ける線引き
- `AI は候補とヒント / 人は観察と判断`

### 追加すべき説明

- なぜ場所が再訪の土台になるのか
- 公開や研究利用で何が追加で必要になるのか
- 普段使う人が最初に守るべき基準

### 書き換え方針

- lead で `場所を残す理由 / 公開範囲を分ける理由 / AI と人の役割 / 見ていないことを言い切らない理由` をまとめて約束する。
- 本文に `使う人が最初に気にすべきこと` を追加して、ルールを実務に落とす。
- `言いすぎない線引き` を最後に置き、前段の理由から自然につながる構成にする。

### 反映後の確認観点

- 1 画面目で `場所 / 公開範囲 / AI と人 / 言いすぎない理由` が取れる。
- 本文で `理由 / 公開ルール / 実際の使い方` が一連で読める。
