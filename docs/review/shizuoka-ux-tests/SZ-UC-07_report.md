## 5. ケース観点ごとの評価

### 良かった点

- **精密入力フィールドの充実**: 環境(biome)、個体数、ライフステージ、地面の状態(substrate)、野生/植栽区分、施設文脈 — モニタリングに必要な要素は概ね揃っている
- **外来種アラート自動表示**: 投稿完了時にinvasiveAlertが自動表示され、外来種の認識を促す
- **外来種フィルタ**: explore.phpで外来種/在来種のフィルタが可能
- **GPS精度表示**: 位置入力時にGPS精度(±Xm)がカラーコード付きで表示される。モニタリングの位置精度管理に有用
- **「前回と同じ」位置復元**: 同一地点の再訪時に位置の再入力を省略可能

### 問題点

- **時系列比較ビューの欠如**: 継続モニタリングの核心機能が不在
- **時刻非表示**: 詳細ページで時刻が消失（P1、横断課題）
- **テンプレート投稿の不在**: 毎回同じ構造の入力を繰り返す負荷
- **外来種判定の透明性不足**: 判定根拠が見えない

### 観察ポイント別メモ

- **精度**: 位置(GPS精度表示あり)、環境、個体数、ライフステージは十分。時刻が表示されない問題あり
- **再訪性**: 「前回と同じ」で位置は復元可能。ただし種名・メモのテンプレートがない
- **研究補助性**: フィールドは充実しているが、蓄積データの分析・比較UIが弱い

## 6. 合格条件チェック

- [x] 位置・日時・対象・メモを無理なく残せる — 入力フィールドは十分 ✓
- [ ] 同地点の継続記録として使える見通しが立つ — **時系列比較ビューがない**
- [ ] 再訪導線や一覧性に大きな不足がない — 複合フィルタ（地点×種名×期間）不可
- [ ] データ品質重視ユーザーが信頼できる印象を持つ — 外来種判定の透明性不足

## 7. 失敗シグナル確認

- 該当した失敗シグナル:
  - **継続比較に必要な情報が埋もれる** — 時系列ビューなし
  - **再検索や一覧導線が弱く、単発投稿にしか見えない** — 複合フィルタ不可
- 該当しなかった失敗シグナル:
  - 精度を上げようとすると入力負荷が急増する ✓
  - 研究補助用途として雑に見える ✓
