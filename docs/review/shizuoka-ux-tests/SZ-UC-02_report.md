## 5. ケース観点ごとの評価

### 良かった点

- **位置保持の設計が秀逸**: `resetForm()` が `lat`/`lng`/`locationSource`/`locationName` をリセットしない。同じ磯で3件連続投稿する際、位置の再入力は不要
- **「もう一枚記録する」ボタン**: 投稿完了画面に明確なCTA。連続投稿への導線が用意されている
- **写真のみで成立**: UC-01と同様、種名不要で写真だけで投稿完了。子連れで名前を調べる余裕がないときに最適
- **EXIF自動抽出の価値**: 磯で撮った写真のGPS・日時が自動入力される。炎天下の磯でフォーム入力する負荷を最小化
- **ゲスト3件**: 3件の連続投稿がゲストのままで完了可能

### 問題点

- **「夏の思い出」として見返せない**: 同日・同場所の記録がフィードで散在。アルバム/トリップビューがない
- **環境(biome)リセット**: 連続投稿時に毎回リセットされ不自然
- **投稿完了画面が使い捨て**: 「もう一枚記録する」で前の投稿情報が消える
- **海辺生物の入力補助不足**: 磯遊びでよく見る生物のプリセットがない

### 観察ポイント別メモ

- **連続投稿**: 位置保持により2件目以降の最大の負荷要因（位置入力）が解消。ただしbiomeリセットは不自然。全体として2件目以降は確実に楽になる設計
- **写真中心UX**: 写真のみで投稿成立。写真プレビューグリッド(2列)も視覚的に分かりやすい。「5枚まで」の上限表示あり
- **子ども連れ利用**: 写真撮影→送信の最短フローは子どもの注意が逸れる前に完了可能。ただし投稿完了画面の情報量が多く（AIメモ待ち、外来種アラート、ネクストアクション）、子どもと一緒に見るには重い
- **思い出としての価値**: 個別の観察詳細は写真+地図で見返せるが、「あの日の磯遊び全部」を一覧で振り返る体験はない。学術データベースとしては正しいが、家族レジャーアプリとしては物足りない

## 7. 失敗シグナル確認

- 該当した失敗シグナル:
  - **投稿後の一覧が断片的で「その日の体験」に見えない** — フィードは個別カード列挙のみ
- 該当しなかった失敗シグナル:
  - 1件目で疲れて2件目に行けない — 位置保持+最短フローで疲労は最小 ✓
  - 海辺生物の扱いが不自然で陸上生物前提に見える — 環境選択に「🌊 海岸・干潟」があり自然 ✓
  - 子どもと一緒に使うには待ち時間や説明量が多すぎる — 最短フローは軽量。ただし投稿完了画面は情報過多気味
