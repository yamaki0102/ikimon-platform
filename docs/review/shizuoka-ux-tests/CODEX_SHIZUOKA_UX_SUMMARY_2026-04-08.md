# Codex 静岡UXケース総括

## 1. 基本情報

- 実施日: 2026-04-08
- 実施者: Codex
- 対象: `SZ-UC-01` 〜 `SZ-UC-10`
- 実施方式: ローカルHTTP確認、コード読解、隔離環境での投稿実証

## 2. ケース別判定

| ケース | 判定 | ひとこと |
| --- | --- | --- |
| SZ-UC-01 | 要改善 | 初心者投稿はできるが、ゲスト導線と位置安心感が弱い |
| SZ-UC-02 | 要改善 | 連投できるが、家族の一日を束ねて見返しにくい |
| SZ-UC-03 | 要改善 | 複数写真投稿は成立するが、同定向け写真整理が弱い |
| SZ-UC-04 | 要改善 | 不明魚を出せるが、投稿後の同定期待形成が弱い |
| SZ-UC-05 | 要改善 | 写真中心は良いが、高齢寄りの現場利用には詳細UIが重い |
| SZ-UC-06 | 要改善 | 生徒個人の楽しさはあるが、教育振り返り導線が弱い |
| SZ-UC-07 | 要改善 | 精度高い記録はできるが、継続比較導線が弱い |
| SZ-UC-08 | 不合格 | 位置秘匿の説明不足と公開座標露出が深刻 |
| SZ-UC-09 | 不合格 | オフライン同期実装が観察投稿と噛み合っていない |
| SZ-UC-10 | 要改善 | 普通種歓迎感はあるが、写真必須で日常軽量投稿に届かない |

## 3. 横断的に強かった点

- 写真中心投稿は全体に強い
- 種名未確定でも観察として残せる
- 投稿成功画面の達成感は良い
- ライフリスト、ストリーク、ウェルネスなど継続要素の種はある

## 4. 横断的な主要課題

- ゲスト導線とログイン導線の役割分担が中途半端
- 位置情報の安心設計が全体に弱い
- オフライン耐性が未完成
- 同定支援への期待形成が投稿後UIで弱い
- 学校・家族・モニタリングなど文脈別モードが不足

## 5. 重大課題

1. 希少種の公開位置まわり
2. オフライン同期先の不整合
3. 写真なし軽量投稿ができないこと
