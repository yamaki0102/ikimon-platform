## 5. ケース観点ごとの評価

### 良かった点

- **複数写真追加がシンプル**: `multiple accept="image/*"` のファイル入力で複数枚一括選択可能。追加後はグリッドプレビューで視覚的に確認できる
- **EXIF自動抽出が全写真に適用**: `handleFiles()` 内で各写真の EXIF を処理し、最適な位置・日時を自動選択する "Multi-Photo EXIF Intelligence"
- **科レベルの投稿が自然に成立**: 種名autocompleteでGBIF/iNatから属・科レベルの候補も返る。入力しなくても投稿可能
- **datetime-local 入力**: 夜間22:10も含めて分単位で入力可能。EXIF日時からの自動プリセットもあり
- **WebP圧縮**: 写真はクライアントサイドでWebP (品質0.72, 最大2560px) に圧縮。高解像度写真の通信負荷を軽減
- **同定エコシステム**: observation_detail.phpに同定提案モーダル、AIアセスメント、コンセンサス表示が実装済み

### 問題点

- **時刻が詳細ページで消失**: 投稿で入力した時分が observation_detail.php で表示されない（P1）
- **写真上限のJSガードなし**: 5枚制限がUIテキストのみ（P2）
- **メイン写真の指定不可**: 同定に最適な写真を選べない（P2）
- **同定信頼度の欠如**: 投稿者の確信度が他ユーザーに伝わらない（P2）
- **メモ欄が小さい**: 詳細な観察メモを書くには不十分（P3）

### 観察ポイント別メモ

- **詳細入力**: datetime-local で22:10の入力は技術的に可能。環境・個体数・ライフステージなど豊富なフィールドがあり、中級者の精密記録ニーズに対応
- **同定保留**: 種名未入力のまま投稿可能で、後日の同定提案を待てる。autocompleteで科レベル候補も返り自然
- **投稿後の専門性**: 写真カルーセル（スワイプ対応）、メモ表示、地図、AI考察、同定タイムラインがあり、同定議論の基盤として概ね十分。ただし**時刻欠落**が致命的

## 7. 失敗シグナル確認

- 該当した失敗シグナル:
  - **時刻やメモの情報密度が足りず、後で使えない** — 時刻が詳細ページで表示されない

  - 写真複数枚の扱いが分かりづらい — グリッドプレビューで直感的 ✓
  - 種名を断定しないと記録価値が下がる印象を与える — 種名任意 + AIアセスメントで価値を補完 ✓
  - 詳細画面が識別支援の文脈に弱い — 同定モーダル + タイムラインで概ね対応 ✓（時刻以外）
