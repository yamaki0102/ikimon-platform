## 5. ケース観点ごとの評価

### 良かった点

- **写真だけで投稿完了**: `canSubmit` は `photos.length > 0` のみを要求。種名・メモ・詳細は全て任意。初心者にとって理想的な最低限設計
- **ゲスト投稿対応**: post.php は未ログインでも3件まで投稿可能。「ゲスト投稿 X/3件」バナーで残数も明示
- **EXIF自動抽出**: 写真から日時・位置を自動取得し「📷 写真から取得」「📷 写真から自動設定」とフィードバック。手動入力の負荷を大幅に削減
- **プログレッシブ・ディスクロージャー**: 詳細フィールドは「名前や詳細を追加する（任意）」ボタンの奥に隠れており、初心者が圧倒されにくい
- **投稿後の達成感**: 「記録された！ 🐾」とconfettiアニメーション、明確なネクストアクション3つ。「キミの気づきが、この場所の生態地図に足あとを刻んだよ」のコピーも秀逸
- **住所検索**: 「住所で検索（例: 静岡市駿河区）」のプレースホルダーで、地図操作に不慣れでも場所を設定できる
- **「撮るだけでOK！名前は後からでも大丈夫」**: 写真エリアのサブテキストが初心者の不安を適切に払拭

### 問題点

- **ボトムナビ → login.php リダイレクト**: 最も目立つ投稿導線がゲストを弾く（P1）
- **エビデンスUI の初心者圧**: 種名を入力した途端に専門的なUIが出現（P2）
- **位置公開粒度の説明不在**: プライバシー不安を解消する情報がフォーム上にない（P2）
- **投稿後のコミュニティ接続が弱い**: 同定支援を受ける明示的な導線がない（P3）

### 観察ポイント別メモ

- **導線**: ヒーローセクションの「生き物を記録する」ボタンからは3秒以内にpost.phpに到達可能 ✓。しかしボトムナビ（モバイルで最も使われるであろう導線）はlogin.phpへリダイレクト ✗
- **入力負荷**: 最短パス（写真→送信）は最小限。ただし「名前や詳細を追加する」を展開すると、日時・種名・エビデンス・野生/植栽・個体由来・施設文脈・地面の状態・ライフステージ・メモ・ライセンスと多数のフィールドが出現。初心者が好奇心で展開した場合の圧は大きい
- **文言**: 写真エリア周辺は平易（「撮るだけでOK」「名前は後からでも大丈夫」）。詳細セクション内は専門的（「同定のエビデンス」「Data Quality」「DwC coordinateUncertaintyInMeters」※後者はコード内のみ）
- **位置情報**: EXIF/GPS自動取得は優秀。住所検索もある。ただし公開範囲の説明が不足
- **投稿後体験**: 達成感は十分。次の行動も明確。ただし「詳しい人に聞く」導線が弱い
- **継続利用性**: 投稿完了後に「📸 もう一枚記録する」ボタンあり。ゲスト制限の先にログインへの自然な誘導もある

## 6. 合格条件チェック

- [x] 3分以内に投稿完了まで到達できる — 写真追加→位置自動取得→送信の最短フローは3タップ設計。3分以内は十分可能（ただし実測ではなくコード構造からの判断）
- [x] 種名の確信がなくても離脱せずに進める — 種名フィールドは任意かつデフォルトで非表示。写真のみで投稿可能
- [ ] 場所公開に対する不安が致命的な離脱要因にならない — 公開粒度の説明がないため、プライバシー意識の高い初心者は不安を感じる可能性あり
- [x] 投稿後、自分の記録が見つけられる — 投稿完了画面に「観察の詳細を見る」リンクあり
- [ ] 初心者でも「また使える」と判断できる — 最短パスでは良い体験。ただしボトムナビからの入口がlogin壁で塞がれている点、種名入力時のエビデンス要求が障壁になりうる

  - 初回体験なのに入力項目が多すぎて面倒に感じる — デフォルトは最小限 ✓（展開時は多い）
