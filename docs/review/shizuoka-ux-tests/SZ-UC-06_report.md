## 5. ケース観点ごとの評価

### 良かった点

- **生物群横断の統一体験**: 海浜植物・鳥・昆虫のいずれでも同じ投稿フロー。分類群による操作の違いがない
- **ゲーミフィケーション基盤が充実**: ポイント(投稿10pt, 同定5pt, 研究用20pt)、バッジ(投稿数・種数・ストリーク等10種類以上)、オブザーバーランク(多軸スコア)、クエストシステム
- **観察会連携の基盤**: post.php に event_id パラメータ連携が既に実装済み
- **レベルアップモーダル**: 投稿完了時にバッジ獲得やクエスト完了がモーダルで祝福される（gamification-modal.js）
- **ストリーク表示**: プロフィールに🔥連続日数が表示され、毎日の記録を動機づける

### 問題点

- **観察会ダッシュボードの不在**: event_id紐づけはあるが、一覧表示画面がない
- **初回体験の地味さ**: 数値ゼロからのスタートで達成感が薄い
- **文言が大人向け**: 高校生には詩的すぎる

### 観察ポイント別メモ

- **初学者導線**: 写真→送信の最短フローは説明不要。「撮るだけでOK」文言あり ✓
- **楽しさ**: バッジ・ランク・ストリーク・クエストで継続動機の仕掛けは十分。ただし初回体験のインパクトが弱い
- **学習利用**: 個人の蓄積は可視化されるが、グループ活動の振り返りが弱い

## 6. 合格条件チェック
