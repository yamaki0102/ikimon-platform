# Content Claims Style Guide

ikimon.life の公開文面で、`GBIF / TNFD / 永続性 / 検証性` を扱うときの表現ルール。

## 目的

- 実装済みの事実と将来構想を混同しない
- 営業・広報・研究向けページで主張の強さを揃える
- 後から根拠を示せない断定表現を避ける

## 基本原則

1. 実装済みの事実だけを断定する
2. 外部連携は、完了前は「見据える」「対応する」「準備する」と書く
3. 制度やフレームワークへの言及は「参照」「整合」「参考」に留める
4. 長期保存は、単独サービスの永続を約束する書き方をしない
5. 品質は「検証可能」「監査可能」に寄せ、「完全」「保証」とは書かない

## 領域別ルール

### GBIF

- 書いてよい
  - Darwin Core / DwC-A 形式で出力できる
  - GBIF などとの連携を見据えた相互運用性がある
  - 共有対象となる場合はライセンス条件を満たす必要がある
- 書かない
  - `GBIF に公開済み`
  - `世界中の研究者がすでに利用している`
  - `GBIF に載るので永続化は完了`

### TNFD

- 書いてよい
  - TNFD LEAP などの外部議論を参照している
  - 社内整理や補助資料の参考に使える
  - 重要判断には専門家レビューや現地確認を推奨する
- 書かない
  - `TNFD の根拠データになる`
  - `TNFD 準拠を証明する`
  - `そのまま開示に使える`

### 永続性

- 書いてよい
  - 長期的に見返せる形で保存する
  - 標準形式で持ち出せる設計を重視する
  - 単独依存を避ける方向で設計している
- 書かない
  - `永久に残る`
  - `100年残ることを保証する`
  - `サービスが停止しても必ず失われない`

### 検証性

- 書いてよい
  - AI評価、コミュニティ同定、レビュー履歴を追える
  - 監査可能な記録に近づける
  - 研究利用ステータスやライセンス条件を明示する
- 書かない
  - `完全に正しい`
  - `誤りがない`
  - `自動で研究品質が保証される`

## 推奨言い換え

- `GBIFに公開する` → `GBIF などとの連携を見据える`
- `TNFDの根拠データ` → `TNFD 検討時の補助資料`
- `永久保存` → `長期的に見返せる形で保存`
- `信頼できるデータ` → `検証履歴を伴うデータ`
- `国際標準に準拠` → `国際標準形式での出力に対応`

## レビュー前チェック

- その文は、コードか運用で今すぐ裏取りできるか
- 連携済みと連携予定を混同していないか
- ユーザーに保証と読まれる表現になっていないか
- 論文や制度への言及が、過剰な権威付けになっていないか
- 重要な但し書きを別ページ送りにしていないか
