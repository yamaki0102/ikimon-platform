# はじめて読む人向けの案内

このページは、ikimon をどの順番で読むと理解しやすいかをまとめた入口です。  
最初から全部を読む必要はありません。自分が今つまずいているところから順に読めば十分です。

## はじめてならこの順番

迷ったら、まず次の 3 ページで全体をつかんでください。

1. [名前が分からないときの基本](/learn/identification-basics)
2. [また同じ場所を見る理由](/learn/field-loop)
3. [記録の扱いと線引き](/learn/methodology)

この 3 ページで、何を残せばよいか、なぜ場所を残すのか、どこまで言い切るのかが分かります。

## 読む順番

読む順番は、困りごとから決めるのが早いです。

- 何を撮ればよいか分からないなら、[名前が分からないときの基本](/learn/identification-basics)
- なぜ同じ場所を見直すのかを知りたいなら、[また同じ場所を見る理由](/learn/field-loop)
- AI や人の判断の違いを知りたいなら、[同定の信頼のしくみ](/learn/authority-policy)
- 公開範囲や言いすぎない線引きを知りたいなら、[記録の扱いと線引き](/learn/methodology)

## 仕組みを知る入口

ikimon の考え方をもう少し知りたい人には、次のページが入口になります。

- [ikimon について](/about)
- [また同じ場所を見る理由](/learn/field-loop)
- [同定の信頼のしくみ](/learn/authority-policy)

ここでは、サービスの目的、AI の役割、任された人の確認がどこで入るかを説明しています。

## 団体利用の入口

学校、地域団体、観察会、地域学習で使えるかを見たい場合は、個人向けの説明を読んだあとで [団体相談](/for-business) を見てください。  
ikimon は、最初から大きな導入を前提にしていません。

特に先に確認してほしいのは次の 3 点です。

- どこで始めたいか
- だれが使うか
- どこまで公開したいか

## 用語で止まったとき

言葉の意味が分からなくなったら、[用語集](/learn/glossary) に戻ってください。  
専門用語を増やすためではなく、読んでいて止まらないための短い補助ページです。
