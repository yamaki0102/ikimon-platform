実装ブロック

  • Header: ロゴ、検索、マイページ入口。長いコピーは置かない。
  • Primary actions: 観察する、同定する、地図、マイページの4つだけ。
  • Topic tabs: おすすめ、近く、同定待ち、地域マップ。読み物はタブではなく棚末尾。
  • Nearby shelf: 写真、候補名、場所丸め、日付、状態ラベル。
  • ID shelf: 同定待ち、追加写真、AI候補を分ける。確認済みと混ぜない。
  • Local map shelf: 再訪問依頼ではなく、地域の変化として見せる。

ルートとKPI

  • `観察する`: `/record`, `landing:topA:primary:record`
  • `同定する`: `/explore?filter=needs_id`, `landing:topA:primary:identify`
  • `地図`: `/map`, `landing:topA:primary:map`
  • `マイページ`: `/profile`, `landing:topA:primary:me`
  • `地域マップ棚`: `/map?focus=local`, `landing:topA:shelf:localMap`

非採用

  • 第1ビューに理念文、制度説明、長いヘルプを置かない。
  • `再訪問してください` を主CTAにしない。
  • ランキング、連続日数、投稿数競争をトップ主役にしない。
  • AI候補を確定名のように見せない。

Senior UI Standards

このワイヤーから実装へ持ち込む、高齢者/低視力/屋外利用向けの最低基準。

項目基準理由
本文16px未満禁止。主要本文は17px、行間1.7。日本語の読み疲れと屋外視認性を下げる。
カード見出し17-20px。1行を基本、長い場合は2行まで。写真を見てから文字で確認できる密度にする。
主要ボタン高さ56px以上。A案の4大入口は76px以上。高齢者と屋外片手操作で誤タップを減らす。
アイコンボタン44px単体は禁止。モバイルでは52-56px。小さい丸アイコンだけの操作を避ける。
チップ/タブ高さ48px以上。隠れた横スクロールだけに依存しない。フィルタを見失わせない。
状態ラベル`AI候補`, `同定待ち`, `確認中`, `追加写真` を文字で併記。色だけで状態を判断させない。
カード情報量写真、見出し、場所/日付、状態ラベルまで。説明文は原則置かない。トップは読む画面ではなく選ぶ画面にする。
余白カード間8-12px、セクション間18-24px以上。密集による誤認識と誤タップを減らす。
文言専門語を先に出さない。`review_ready` は `確認しやすい記録` に変換。内部品質語を利用者へ直出ししない。
地図トップでは操作地図より要約地図。詳細操作は `/map` へ送る。トップの重さと誤操作を抑える。
