# 俺の屍を超えてゆけ — Ai 世代継承成長プロトコル

> **コンセプト**: ゲーム「俺の屍を超えてゆけ」のように、各会話は1つの「世代」。
> 世代が終わるたびに学びを結晶化し、次の世代（会話）に継承する。
> 子孫はどんどん強くなる。

## 発動タイミング

### 自動発動 (ALWAYS)
- **会話の終了前**（大きなタスクを完了したとき）
- **重大な失敗から回復したとき**
- **新しいパターンを発見したとき**

### 手動発動
- `/oreshika` — ユーザーが手動で成長サイクルを開始

## 成長サイクル（4ステップ）

### Step 1: 戦闘記録（Battle Log）
この会話で何があったかを**事実ベース**で振り返る。

```
振り返りポイント:
- 何を実装/修正/調査したか
- どこで詰まったか（3-Strike Ruleに引っかかったか）
- どの推測が外れたか
- 何が予想外にうまくいったか
- キミ（ユーザー）から学んだ洞察は何か
```

### Step 2: 経験値抽出（XP Extraction）
戦闘記録から**再利用可能な知識**を抽出する。

| 種類 | 説明 | 例 |
|---|---|---|
| **⚔️ 戦技** | 具体的なコード/コマンドパターン | `DataStore::save()` は `flock()` を使うので同時書き込みに注意 |
| **🛡️ 防御** | ハマったトラップと回避法 | サーバーの `public_html/` 直下に置いても古いファイルが配信される罠 |
| **🧠 戦略** | 設計判断・アーキテクチャ上の学び | 教典にKPIがないと「完成したか」を判定できない |
| **💬 対話** | ユーザーとのコミュニケーション上の気づき | キミは「ガンガンやれ」と言ったら本当に任せてくれる |
| **🔮 予見** | 将来必要になりそうな知識 | Phase I でJWT移行する時にセッション管理を全面置換する必要がある |

### Step 3: 継承結晶（Heritage Crystal）
抽出した経験値をMemory/KIに書き込む。

**書き込み先の判断基準:**
| 対象 | 書き込み先 | 条件 |
|---|---|---|
| プロジェクト固有の知識 | `GEMINI.md` に追記 or 既存KIを更新 | そのプロジェクトでのみ有効 |
| 横断的なパターン | `ikan_group_technical_standards` KI 更新 | 複数プロジェクトで有効 |
| Ai自身の行動原則 | `user_global` Memory 更新提案 | キミの承認が必要 |

**書き込みフォーマット（Anchored Iterative Summarization）:**

> 構造がないと情報は静かに消える。各セクションは「チェックリスト」として機能し、
> 要約者が必ず埋めなければならない。これにより世代間の情報損失を防ぐ。

```markdown
## 世代 [会話ID短縮] — [日付]

### セッション意図
[この会話でユーザーが達成しようとしていたこと]

### 獲得経験値
- ⚔️ [戦技の内容]
- 🛡️ [防御の内容]
- 🧠 [戦略の内容]

### 変更したファイル
- [file.php]: [何を変更したか]
- [file2.php]: [読んだが変更なし]

### 判断の根拠
- [なぜその方法を選んだか、却下した代替案は何か]

### 現在の状態
- [テスト結果、残タスク、ブロッカー]

### 次の世代への申し送り
1. [最優先で確認すべきこと]
2. [継続すべき作業]
```

> [!TIP]
> **tokens-per-task 原則**: 圧縮率を追求するあまり詳細を失うと、次の世代が
> 再調査に時間を費やす。0.7%の追加トークンが0.35ポイントの品質向上を生む。
> 薄い圧縮 > 深い損失。

### Step 4: 進化報告（Evolution Report）
キミに報告する。簡潔に。

```
🧬 世代継承完了
━━━━━━━━━━━━
⚔️ 戦技 +2  🛡️ 防御 +1  🧠 戦略 +3
💬 対話 +1  🔮 予見 +1

📝 継承先: GEMINI.md, ikimon_biodiversity_platform KI
━━━━━━━━━━━━
次の世代の私はこれを知っている。
```

## 成長の可視化

各プロジェクトのGEMINI.mdに「世代ログ」セクションを追加する:

```markdown
## 世代ログ（Oreshika Heritage）
| 世代 | 日付 | 戦技 | 防御 | 戦略 | 対話 | 予見 | 主な学び |
|---|---|:---:|:---:|:---:|:---:|:---:|---|
| Gen-1 | 2026-02-08 | +2 | +1 | +3 | +1 | +1 | 教典にKPI/スキーマ必須、OAuth PKCE |
| Gen-2 | ... | ... | ... | ... | ... | ... | ... |
```

## 注意事項

- **真実優先**: 「学んだ」と書くなら証拠を残す。誇張禁止
- **量より質**: 10個の薄い学びより1個の深い学びを継承する
- **失敗こそ財産**: 「3回失敗して判明した」は最高の防御知識
- **キミの言葉を引用する**: ユーザーの洞察は最も価値が高い。そのまま記録する
