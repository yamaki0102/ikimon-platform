# サイトマップ & 動線設計プロトコル

## いつ発動するか

以下のいずれかに該当する作業を行った**直後**に、このスキルの手順を実行すること。

- 新しい PHP ページを作成した
- 既存ページを削除した
- ページの URL パスを変更した
- ナビゲーション（nav.php / footer.php）を変更した
- ページ間のリンク構造を大きく変更した

## 手順

### Step 1: サイトマップ更新

サイトマップは以下の場所に保管されている:

```
.gemini/antigravity/brain/<conversation-id>/sitemap.md
```

> [!IMPORTANT]
> sitemap.md が現在の会話に存在しない場合は、最新の KI（`ikimon_biodiversity_platform`）または最近の会話の sitemap.md をベースに新規作成すること。

#### 更新内容チェックリスト

- [ ] **MECE 6分類テーブル** にページを追加/削除
  - 1.パブリック / 2.観察・投稿 / 3.同定・図鑑 / 4.サイト・レポート / 5.ユーザー / 6.管理
- [ ] 各テーブルの `#` 番号を振り直し
- [ ] MECE 検証テーブルのページ数を更新
- [ ] API 一覧を更新（API を追加/削除した場合）

### Step 2: 動線マップ更新 ← **最重要**

> [!CAUTION]
> **動線設計を忘れずに！** ページを追加しただけでは誰もたどり着けない「孤立ページ」になる。

#### 新規ページ追加時

1. **どこからリンクするか** を決める:
   - グローバルナビ（nav.php トップバー）
   - ユーザーメニュー（nav.php ドロップダウン）
   - モバイルボトムナビ（nav.php 5タブ）
   - フッター（footer.php）
   - 親ページからのコンテキストリンク（例: events.php → event_detail.php）
   - LP・特定ページからの CTA

2. **リンクを実装する** — 該当するナビゲーションファイルを編集

3. **動線マップを更新する** — sitemap.md の「動線マップ」セクション:
   - 該当するユーザージャーニー図（Mermaid）にノードを追加
   - 動線サマリーテーブルに行を追加
   - 孤立ページリストから削除（該当する場合）

#### ページ削除時

1. 削除ファイルへの参照を検索:
   ```powershell
   findstr /s /i "ファイル名" public_html\*.php
   ```
2. 参照がある場合はリンクを除去
3. sitemap.md から該当ページを削除
4. 動線マップの Mermaid 図からノードを削除

### Step 3: 孤立チェック

新規ページを追加した後、以下を必ず確認:

| チェック項目 | 確認方法 |
|---|---|
| Desktop からアクセス可能？ | nav.php のリンク or 親ページからの遷移 |
| Mobile からアクセス可能？ | ボトムナビ or ユーザーメニュー or 親ページ |
| 検索エンジンから発見可能？ | sitemap.php に含まれるか |

**3つすべて ✗ の場合、そのページは孤立ページ。** リンクを追加するまで作業を終えないこと。

### Step 4: ナビゲーション構造ルール

#### nav.php トップバー（Desktop）
- 最大5項目まで（現在: explore, compass, events + CTA）
- これ以上追加する場合はユーザーメニュー配下に入れる

#### モバイルボトムナビ（5タブ固定）
- home, explore, **post（中央CTA）**, zukan, profile
- **変更禁止** — 5タブは認知負荷の限界

#### ユーザーメニュー
- ログイン必須ページや中級者以上向けの機能を配置
- 役割制限チェック（`Auth::hasRole()`）で管理者リンクを出し分け

#### フッター
- 静的情報ページ（about, terms, privacy 等）
- B2B リンク（for-business）
- 追加は自由だが、8項目以下を推奨

## 関連ファイル

| ファイル | 役割 |
|---|---|
| `components/nav.php` | グローバルナビ + ユーザーメニュー + ボトムナビ |
| `components/footer.php` | フッターリンク |
| `sitemap.php` | XML サイトマップ生成（SEO） |
