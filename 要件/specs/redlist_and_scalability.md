# 希少種・レッドリスト対応設計書
## 地域・県・国・国際の階層構造を適切に扱う

---

## 1. 日本のレッドリスト階層構造

### 1.1 複雑な階層

```
[国際] IUCN Red List
    ↓
[国] 環境省レッドリスト/レッドデータブック
    ↓
[都道府県] 都道府県版レッドリスト（47種類）
    ↓
[地域] 市町村版（一部地域のみ）
```

### 1.2 階層ごとの特徴

| レベル | 発行元 | 更新頻度 | 特徴 |
|--------|--------|---------|------|
| **IUCN** | 国際自然保護連合 | 随時 | グローバル基準、分布域全体で評価 |
| **環境省** | 環境省 | 5年程度 | 日本国内での状況、法的根拠なし（参考資料） |
| **都道府県** | 各都道府県 | 5-10年 | **県内での状況**（同じ種でも県ごとに異なる） |
| **市町村** | 一部自治体 | 不定期 | 限定的、データ入手困難 |

### 1.3 カテゴリの違い（環境省）

| カテゴリ | 略称 | 意味 |
|---------|------|------|
| 絶滅危惧IA類 | CR | ごく近い将来に絶滅の危険性が極めて高い |
| 絶滅危惧IB類 | EN | 近い将来に絶滅の危険性が高い |
| 絶滅危惧II類 | VU | 絶滅の危険が増大 |
| 準絶滅危惧 | NT | 現時点では絶滅危険度は低いが注意 |
| 情報不足 | DD | 評価するだけの情報がない |
| 絶滅のおそれのある地域個体群 | LP | 地域的に孤立した個体群 |

## 2. ikimonでの実装方針

### 2.1 マルチレベル表示

**同じ種に複数のステータスが存在する場合がある。**

```
[アカガエル科の某種の例]

📍 浜松市で観察

レッドリストステータス:
├─ 🌍 IUCN: LC（低懸念）
├─ 🇯🇵 環境省: VU（絶滅危惧II類）
└─ 📍 静岡県: CR（絶滅危惧IA類）← 県内では深刻

⚠️ この観察地点では「静岡県CR」が最も重要な指標です
```

### 2.2 表示優先度ロジック

```
観察地点の都道府県を特定
    ↓
該当都道府県のレッドリストを最優先で表示
    ↓
環境省レッドリストを補足表示
    ↓
IUCNは参考として表示
```

### 2.3 位置秘匿ルール（マルチレベル対応）

| 判定基準 | 秘匿レベル |
|---------|----------|
| **県レッドリストで CR/EN** | 10kmグリッドに丸め |
| **県レッドリストで VU** | 1kmグリッドに丸め |
| **環境省レッドリストのみ該当** | 1kmグリッドに丸め |
| **IUCNのみ該当** | フル精度（日本では珍しくない可能性） |
| **いずれにも該当しない** | フル精度 |

## 3. データ同期自動化システム

### 3.1 階層別自動化レベル

| レベル | 自動化度 | 方法 |
|--------|---------|------|
| **IUCN** | ⭐⭐⭐ 完全自動 | API（公式提供） |
| **環境省** | ⭐⭐ 半自動 | Webスクレイピング or 公開CSV |
| **都道府県** | ⭐ 半手動 | 年1回一括取得 + 差分検知 |
| **市町村** | 手動 | 入手時に追加 |

### 3.2 IUCN（完全自動）

```
[週次バッチ]
1. IUCN Red List API v4 から日本産種を取得
   GET /api/v4/country/getspecies/JP
2. 既存マスタと比較
3. 差分があれば更新 + 管理者通知
4. ログ保存
```

**API情報**:
- エンドポイント: `https://apiv4.iucnredlist.org/`
- 認証: トークン必要（無料申請可）
- レート制限: あり（バッチで対応可能）

### 3.3 環境省（半自動）

```
[月次バッチ]
1. 環境省サイトをスクレイピング
   https://www.env.go.jp/nature/kisho/hozen/redlist/
2. PDFがあればOCR処理
3. または公開CSVを取得（あれば）
4. GBIF Backboneで学名を正規化
5. 既存マスタと比較
6. 差分があれば「要確認」としてキューに追加
7. 管理者が確認後に反映
```

**課題と対策**:
| 課題 | 対策 |
|------|------|
| PDFで公開されることが多い | OCR + 手動確認 |
| 更新頻度が5年ごと | 月次でチェックし、変更検知時に通知 |

### 3.4 都道府県（半手動 + 差分検知）

```
[年次処理]
Phase 1: 自動スクレイピング
├─ 各県の環境部局サイトをクロール
├─ レッドリスト関連ページを検出
└─ 更新日時を取得

Phase 2: 変更検知
├─ 前回取得日より新しい場合 → アラート
└─ 管理者がダウンロード・整形

Phase 3: データ整形
├─ Excel/PDF → CSV変換（手動 or スクリプト）
├─ GBIF Backboneで学名正規化
└─ マスタに反映
```

**47都道府県対応表の管理**:
```json
{
  "prefectures": [
    {
      "code": "22",
      "name": "静岡県",
      "redlist_url": "https://www.pref.shizuoka.jp/.../redlist",
      "last_checked": "2024-12-01",
      "last_updated": "2023-03-15",
      "format": "excel",
      "automation_level": "semi-auto"
    },
    ...
  ]
}
```

### 3.5 学名・和名の名寄せシステム（Name Reconciliation）

**問題**: 都道府県レッドリストはGBIF準拠ではない
- 和名表記の揺れ（カタカナ/ひらがな/漢字）
- 学名の古い表記（シノニム）
- 独自の亜種・変種の扱い
- 方言名の混入

**解決策**: 多段階マッチングパイプライン

```
[入力: 都道府県レッドリストの1行]
「ニホンアカガエル Rana japonica 絶滅危惧II類」
    ↓
Stage 1: 完全一致チェック
├─ 和名 → ikimonマスタ → ヒット → 終了
└─ 学名 → GBIFマスタ → ヒット → 終了
    ↓ (ヒットなし)
Stage 2: GBIF Species Match API
├─ POST /v1/species/match?name=Rana japonica
├─ レスポンス: {"matchType": "EXACT", "usageKey": 2427091, ...}
└─ acceptedName取得 → マスタに追加
    ↓ (matchType = "FUZZY" or "HIGHERRANK")
Stage 3: シノニム検索
├─ GBIF Backbone でシノニム一覧取得
├─ 過去の学名でマッチ → 現在の有効名に変換
└─ 例: Rana japonica → Rana japonica (accepted)
    ↓ (マッチなし)
Stage 4: ファジーマッチング
├─ Levenshtein距離 ≤ 2 の候補を抽出
├─ 和名の部分一致（「アカガエル」含む）
└─ 候補リストを生成
    ↓
Stage 5: 人間レビューキュー
├─ 自動マッチできなかった種を管理画面に表示
├─ 管理者が正しいマッピングを選択
└─ 学習データとして蓄積 → 次回以降は自動マッチ
```

**マッチング結果の保存**:
```json
{
  "source": "shizuoka_redlist_2023",
  "original_name": "ニホンアカガエル",
  "original_scientific": "Rana japonica",
  "matched_taxon_key": 2427091,
  "matched_name": "Rana japonica",
  "match_type": "exact",  // exact, fuzzy, synonym, manual
  "confidence": 1.0,
  "reviewed_by": null  // or "admin_yamaki"
}
```

**学習データベース**:
```
[過去のマッピング実績]
「にほんあかがえる」→ Rana japonica (信頼度: 1.0)
「日本赤蛙」→ Rana japonica (信頼度: 1.0)
「Rana temporaria japonica」→ Rana japonica (シノニム)
```

**エッジケース処理**:

| ケース | 処理 |
|--------|------|
| **亜種・地域型** | 県独自の「北陸型」など → 種レベルにフォールバック + フラグ |
| **複数和名** | 全て学習DBに登録（1対多マッピング対応） |
| **地域個体群 (LP)** | 種とは別にLPとして独立登録 |
| **未記載種・〇〇の一種** | 上位分類群にマッピング + `is_undescribed: true` |
| **外来種（和名なし）** | 学名のみでマッチ、和名は空欄許容 |

## 4. グローバル展開対応

### 4.1 設計思想

> **最初は浜松、でも設計は世界対応**

| 項目 | ローカルスタート | グローバル対応 |
|------|----------------|---------------|
| **言語** | 日本語のみ | UIはJSON多言語対応済み |
| **地図** | 地理院タイル | OpenStreetMap/Mapboxに差し替え可能 |
| **種DB** | GBIF（世界対応済み） | そのまま使える |
| **レッドリスト** | 日本47都道府県 | 国別 + 地域別に拡張可能な構造 |
| **法規制** | 日本の法律参照 | 国別法規制DB追加可能 |

### 4.2 地域構造の抽象化

```
[現在: 日本専用]
Japan
├─ Prefectures (47)
│   └─ Municipalities
└─ Red Lists: IUCN → 環境省 → 都道府県

[将来: グローバル]
World
├─ Countries
│   ├─ Japan
│   │   └─ Prefectures → Municipalities
│   ├─ UK
│   │   └─ Countries (England, Wales, Scotland, NI) → Counties
│   ├─ Australia
│   │   └─ States → Local Government Areas
│   └─ ...
└─ Red Lists: IUCN → National → Regional
```

### 4.3 地域追加時のチェックリスト

新しい国/地域を追加するとき:

- [ ] 地図タイルソースの確認
- [ ] 行政区画データ（GeoJSON）の入手
- [ ] 国レベルレッドリストの入手・整形
- [ ] 地域レベルレッドリストの入手（あれば）
- [ ] 希少種の法規制確認（種の採取禁止など）
- [ ] 言語JSONの追加
- [ ] 通貨対応（有料プランがある場合）

## 5. 法規制との連携

### 5.1 日本の関連法規

| 法律 | 対象 | ikimonでの対応 |
|------|------|---------------|
| **種の保存法** | 国内/国際希少野生動植物種 | 位置完全秘匿 + 警告表示 |
| **文化財保護法** | 天然記念物 | 情報表示 |
| **鳥獣保護法** | 狩猟鳥獣 | 参考情報表示 |
| **外来生物法** | 特定外来生物 | アラート表示 |

### 5.2 法規制マスタ

```json
{
  "taxonKey": 789,
  "scientificName": "Nipponia nippon",
  "vernacularName": "トキ",
  "legalStatus": {
    "japan": [
      { "law": "種の保存法", "category": "国内希少野生動植物種" },
      { "law": "文化財保護法", "category": "特別天然記念物" }
    ]
  },
  "locationPrivacy": "full_obscure",  // 位置情報完全非公開
  "warningMessage": "この種は「種の保存法」により保護されています。位置情報は非公開です。"
}
```
