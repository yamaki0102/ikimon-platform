# 生物多様性クレジット対応設計書
## Biodiversity Credit Readiness Requirements

---

## 1. 生物多様性クレジットとは

### 1.1 定義
- **カーボンクレジットの生物多様性版**
- 検証可能・定量化可能・取引可能な「自然へのポジティブな成果」の単位
- オフセット（相殺）ではなく**ネットポジティブ（純増）**を目指す

### 1.2 国際動向

| 地域/組織 | 状況 |
|----------|------|
| EU | タクソノミー規制で生物多様性開示義務化への動き |
| UK | 環境法でBiodiversity Net Gain (BNG)義務化（2024年〜） |
| オーストラリア | Nature Repair Market Act (2023) |
| コロンビア | 生物多様性クレジット試験取引開始 |
| 日本 | **未導入**（TNFD開示は進行中） |

### 1.3 主要フレームワーク

| 名称 | 特徴 |
|------|------|
| **Verra SD VISta Nature Framework** | Quality Hectares (Qha) を単位 |
| **Plan Vivo PV-Nature** | コミュニティベース、5年ごと検証 |
| **BioCarbon Standard** | カーボンと生物多様性の統合 |
| **Biodiversity Credit Alliance (BCA)** | 高整合性原則を策定 |

## 2. クレジット対応に必要な要件

### 2.1 高整合性原則（BCA準拠）

| 原則 | ikimonでの対応 |
|------|---------------|
| **追加性 (Additionality)** | プロジェクト登録時点からのベースライン記録 |
| **永続性 (Durability)** | 最低20年のデータ保持・追跡可能性 |
| **漏出防止 (No Leakage)** | エリア外への影響モニタリング |
| **検証可能性 (Verifiability)** | 第三者検証対応のデータ構造 |
| **透明性 (Transparency)** | 全データのオープンアクセス |

### 2.2 データ要件

| 項目 | 仕様 | 理由 |
|------|------|------|
| **位置精度** | GPS精度10m以内推奨 | 正確なエリア紐付けに必須 |
| **時系列記録** | タイムスタンプ+不変性保証 | 経年変化の証明 |
| **種同定品質** | Research Grade必須 | 科学的検証可能性 |
| **観察者ID** | 一意識別可能 | トレーサビリティ |
| **メタデータ** | Darwin Core完全準拠 | 国際互換性 |

### 2.3 MRV（監視・報告・検証）対応

```
M: Monitoring（監視）
├─ 定点観測サポート（同一地点の経年比較）
├─ 種リスト自動生成（時期別）
└─ 外来種アラート

R: Reporting（報告）
├─ TNFD対応レポート出力
├─ SBTN (Science Based Targets for Nature) 準拠
└─ 年次変化サマリー

V: Verification（検証）
├─ Research Grade = 2人以上の合意
├─ 認定研究者レビュー
└─ 第三者監査用エクスポート
```

## 3. ikimonに追加すべき機能

### 3.1 ベースライン設定機能

```
[サイト登録時]
1. エリア境界を描画
2. 登録日時点の種リストをスナップショット保存
3. これが「ベースライン」（基準点）となる

[経年比較]
Year 0 (Baseline): 50種確認
Year 1: 55種確認 → +5種（クレジット化可能な成果）
Year 2: 58種確認 → +3種
```

### 3.2 Quality Hectares (Qha) 対応

Verra SD VISTa 準拠の単位計算:

```
Qha = エリア面積(ha) × 生態系条件スコア(0-1)

生態系条件スコア = f(
  種多様性指数,
  希少種存在率,
  外来種不在率,
  生息地連続性
)
```

**ikimon実装**:
- 種多様性: Shannon-Wiener指数から算出
- 希少種: レッドリスト種のカウント
- 外来種: 特定外来生物のフラグ
- 連続性: 隣接サイトとの接続（将来拡張）

### 3.4 第三者検証対応エクスポート

## 4. TNFD対応レポート出力

### 4.1 TNFDが求める開示項目

| カテゴリ | 開示項目 | ikimonからの出力 |
|---------|---------|-----------------|
| **ガバナンス** | 自然関連リスクの監督体制 | （企業側で記述） |
| **戦略** | 事業への影響評価 | サイト別インパクト分析 |
| **リスク管理** | 特定・評価プロセス | 外来種アラート、変化トレンド |
| **指標・目標** | 定量的な目標と進捗 | **種数、Qha、Research Grade率** |

```

「クレジット対応可能なデータを提供するプラットフォーム」になる

[価値提案]

```
