# 第1章: サービス哲学

## 1.1 ポジショニング：「第三の道」

| サービス | アイデンティティ | 致命的弱点 |
|---------|----------------|-----------|
| **Biome** | 「自然のソシャゲ」（AI-first Game） | 科学者から信頼されない |
| **iNaturalist** | 「研究者のためのツール」 | 一般ユーザーに敷居が高い |
| **いきものログ** | 「環境省のDB」 | UXが死んでいる |
| **ikimon** | **「地域の自然インフラ」** | ← ここを狙う |

## 1.2 核心思想

### 観察は「公共財」である
- ポイントでもバッジでもなく、**保全活動・地域政策・金融価値（クレジット）**への貢献可能性
- 「正解を当てる」のではなく「みんなで解明する」

### 人間ファースト
- AIによる種同定は**しない**（Phase 1）
- 専門家と市民の対話が価値の中心
- 争いは「見せない」、システムで静かに解決

### ハイパーローカル
- 最初は「浜松」などの特定地域に特化（**設計は全世界対応**）
- リアルの場（公園・看板）と深く連携（O2O）

# 第2章: 心理学的設計基盤

## 2.1 自己決定理論（Deci & Ryan）

| 要素 | 意味 | ikimonでの実装 |
|------|------|---------------|
| **自律性** | 「自分で選んでいる」感覚 | 同定の強制なし、好きな分類群だけ見れる |
| **有能感** | 「できた！」という達成感 | 段階的達成、博士バッジ |
| **関係性** | 「繋がっている」帰属感 | 専門家フィードバック、地域コミュニティ |

## 2.2 自己効力感（Bandura）

| 要素 | 実装 |
|------|------|
| **達成経験** | 最初は簡単な観察でResearch Gradeを経験させる |
| **代理経験** | 他ユーザーの成功事例（「今週のベスト同定」）表示 |
| **言語的説得** | 専門家からの「素晴らしい観察です」コメント |
| **情動喚起** | ストレスを与えない（間違っても罰なし） |

## 2.3 エラーを責めない設計

❌ 「同定が間違っています」
✅ 「別の見方もあるようです。確認してみませんか？」

# 第3章: ユーザー体験設計

## 3.1 ユーザー階層

| 階層 | 取得方法 | 権限 |
|------|---------|------|
| **ゲスト** | 自動 | 投稿のみ（3件まで） |
| **一般ユーザー** | LINE/Googleログイン | 投稿、同定投票、ライフリスト |
| **アクティブ同定者** | 自動昇格（30日10件同定） | 同定重み+1 |
| **分類群博士** | 自動認定（上位5%） | 同定重み+3、バッジ表示 |
| **認定研究者** | **IKIMON社審査制** | 同定重み+5（専門分類群のみ） |

## 3.2 認証フロー

```
1. QRからPWA起動
2. ログインなしで写真投稿（最大3件）
3. 後からLINE/Google連携で「投稿を引き継ぎますか？」
4. 紐付け完了 → ライフリストに追加
```

## 3.3 体験モード（MECE）

1. **探索モード** - 身近な自然を発見し親しみを持つ
2. **支援モード** - 支援すべきサイトを見つけ資源を提供
3. **申請モード** - 所有地の認定確認、申請準備
4. **分析モード** - 地域の保全状況把握、データ活用
5. **専門家モード** - 同定センターで「千本ノック」

# 第4章: 同定ロジック設計

## 4.1 重み付き投票システム

```
投票の重み = log₂(信頼スコア + 1) + 役割ボーナス
```

| 役割 | ボーナス | 条件 |
|------|---------|------|
| 一般ユーザー | +0 | デフォルト |
| アクティブ同定者 | +1 | 過去30日に10件以上同定 |
| 分類群博士 | +3 | 該当分類群で上位5% |
| 認定研究者 | +5 | **専門分類群のみ適用** |

> **重要**: 鳥類学者が昆虫を同定しても+5は付かない。

## 4.2 同定ランク（種だけではない）

| ランク | Research Grade可否 |
|--------|-------------------|
| **種** | ✅ 可能 |
| **属** | ✅ 同定難易度が高い場合は可 |
| **科** | ⚠️ 条件付き（写真不鮮明時） |
| **目以上** | ❌ 不可（Best Effortとして記録） |

## 4.3 ステータス

| ステータス | アイコン | 意味 |
|-----------|---------|------|
| **Needs ID** | 🔍 | 名前を教えてくれる人を待っています |
| **Suggested** | 💬 | 候補が出ました |
| **Agreed** | ✅ | みんなが同意しています |
| **Research Grade** | 🏅 | 科学に貢献しました！ |
| **Best Effort** | 📸 | 分かる範囲で最善を尽くしました |
| **Expert Review** | 🔬 | 専門家がレビュー中 |

## 4.4 時間減衰と再検証

| トリガー | 再検証フラグ |
|---------|------------|
| 分類改訂（GBIF検知） | ✅ 発動 |
| 認定研究者の異議 | ✅ 発動 |
| 主要同定者の信頼崩壊 | ✅ 発動 |
| **時間経過のみ** | ❌ **発動しない** |

# 第5章: 紛争解決設計

## 5.1 争いを「見せない」設計

> **原則**: 争いが見えるとユーザーは離脱する。

| 設計 | 実装 |
|------|------|
| **公開コメント欄なし** | 議論スレッドを作らない |
| **同定は投票のみ** | 種名を選ぶだけ。言葉のやり取りなし |
| **異議は非公開** | 提案者と管理者のみ見える |
| **解決後のみ表示** | 議論中は「確認中」と表示 |

## 5.2 「負けた」側へのケア

| ケア | 説明 |
|------|------|
| 意見は永続記録 | 将来の分類改訂時に参照される |
| 貢献ポイント付与 | 議論参加自体が評価される |
| 却下通知なし | 「却下されました」は送らない |
| 将来の復権 | 少数派が正しかったと判明した場合「先見の明」として称える |

## 5.3 Expert Override

| フェーズ | 待機期間 |
|---------|---------|
| ローンチ期（研究者<10人） | **無期限** |
| 成長期（10-50人） | **14日** |
| 安定期（>50人） | **7日** |

# 第7章: データ標準（Darwin Core準拠）

## 7.1 必須フィールド

| DwCフィールド | 取得方法 |
|--------------|---------|
| `occurrenceID` | `ikimon:obs:UUID` |
| `eventDate` | Exif or 手動 |
| `scientificName` | GBIF Backbone |
| `vernacularName` | 種マスタ |
| `decimalLatitude/Longitude` | GPS |
| `coordinateUncertaintyInMeters` | GPS精度 |
| `geodeticDatum` | 固定: WGS84 |

## 7.2 希少種位置秘匿

| レッドリスト | 公開精度 | 表示方法 |
|------------|---------|----------|
| CR / EN | 10kmグリッド内の**ランダム点** | 中心には見せない |
| VU | 1kmグリッド内の**ランダム点** | 同上 |
| NT / LC | フル精度 | 実際の位置 |

> **重要**: グリッドの「中心」に表示すると密猟者に場所を特定される。
> ランダム化して「この辺りのどこか」という印象にする。

## 7.3 エクスポート形式

- Darwin Core Archive (DwC-A)
- CSV (Simple DwC)
- GeoJSON
- JSON-LD (Schema.org)

# 第9章: 運用設計

## 9.1 写真管理

### 複数写真対応
- **1観察 = N枚の写真**（全体像、アップ、特徴部）
- 最初の1枚がサムネイル表示
- 各写真にメモ追加可能

### EXIF自動読み取り
- 日時: Exif DateTimeOriginal → 観察日時に自動入力
- 位置: Exif GPS → 地図に自動プロット
- 複数写真の場合: 最初の写真のExifを採用、差異があれば確認

### 自動圧縮（クライアント側）
| 項目 | 仕様 |
|------|------|
| 入力 | ユーザーの元写真（何MBでもOK） |
| 処理 | Canvas APIでリサイズ + WebP変換 |
| 出力 | **1280px / WebP / 500KB以下** |
| 著作権 | ユーザー保有、ikimonにライセンス付与 |

> ユーザーは元ファイルサイズを気にする必要なし。システムが自動対応。

## 9.2 投稿時の野生/植栽判定

| 選択肢 | デフォルト | 説明 |
|---------|----------|------|
| 野生 | ✅ | 自然に生育している個体 |
| 植栽/飼育 | | 人が植えた・飼っている |
| 不明 | | 判断できない場合 |

- 植栽/飼育は Research Grade 対象外（陸域のみ）
- でも投稿自体は歓迎（記録として価値あり）
- 「植えたけど野生化した」ケースは「野生」扱い

## 9.3 モデレーション

| 対策 | 実装 |
|------|------|
| NSFW自動検出 | 画像フィルタAPI |
| 連続投稿制限 | 1分5件で警告、10件で停止 |
| 同一画像検出 | ハッシュ比較 |
| シャドウバン | 問題ユーザーは他者に非表示 |

**問題ユーザーの定義**:
- スパム投稿（同じ写真の連続投稿）
- 不適切コンテンツ（NSFW、無関係な写真）
- 荷らし行為（他ユーザーへの嫌がらせ）
- 悪意のある誤同定（わざと間違える）

> 対応: 警告 → 一時停止 → シャドウバン → 永久BAN

## 9.4 通知設計

| イベント | デフォルト | 心理効果 |
|---------|----------|---------|
| 同定完了 | ✅ ON | 知的好奇心充足 |
| Research Grade達成 | ✅ ON | 有能感・貢献感 |
| 博士バッジ獲得 | ✅ ON | 承認欲求 |

## 9.4 オフライン対応

- 写真撮影・保存はローカルで完結
- オンライン復帰時に同期

## 9.5 多言語対応

- 初期: 日本語 + 英語
- UIラベルはJSONから取得

# 第11章: 収益化モデル

## 11.1 年間サブスクリプション

### 直接契約（エンドユーザー）

| サイト数 | 年額（税別） |
|---------|------------|
| 1サイト目 | **¥100,000** |
| 2サイト目以降 | ¥50,000/サイト |

### 代理店アカウント

| サイト数 | 年額（税別） |
|---------|------------|
| 1サイト目 | **¥100,000** |
| 2サイト目以降 | **¥50,000/サイト** |

**代理店メリット**:
- 複数クライアントを1アカウントで管理
- 2サイト目以降が割安 → **代理店経由の方がお得**
- 代理店向けダッシュボード（全サイト一覧）
- 請求書一括発行

### 価格設計の意図

```
[ロジック]
多くの企業・団体は自然共生サイトを1件しか持たない
    ↓
直接契約だと2件目のメリットを活かせない
    ↓
「代理店を通した方がお得」という構造
    ↓
代理店がサービス拡大の鍵となる
```

**代理店例**:
- 環境コンサルタント
- OECM申請支援事業者
- 自治体の環境部門
- 地域の自然保護団体

## 11.2 サブスクに含まれる機能

年間¥100,000のサブスクに以下が全て含まれる:

- ✅ **CSRショーケースページ**
- ✅ **イベント作成機能**（観察会の作成・参加管理）
- ✅ **OECM申請レポート**（種リスト自動出力）
- ✅ 種リストCSVエクスポート
- ✅ 担当者連絡先編集

## 11.3 CSRショーケースページ（詳細）

**目的**: 新卒説明会・地域説明会で「ネイチャーポジティブの取り組み」を1画面で伝える。

### 表示セクション

| セクション | 内容 |
|--------------|------|
| **ヒーロー** | 企業名 + キャッチコピー + 代表写真 |
| **数字サマリー** | 確認種数 / Research Grade件数 / 参加者数 |
| **活動エリアマップ** | サイト境界 + 観察ヒートマップ |
| **最近の発見** | 新着観察3件（写真付き） |
| **ハイライト種** | 希少種・特徴的な種（写真付き） |
| **生物多様性スコア** | ★★★★☆ 形式で視覚化 |
| **イベント履歴** | 過去の観察会一覧 |

### embed対応
```
<iframe src="https://ikimon.life/showcase/company-id" />
```
企業サイトや採用ページに埋め込み可能。

**効果**: 担当者の自己効力感向上 + 採用活動での差別化
