# Bio-Navigator: Role & Philosophy Definition

The user defined the vision: **"Akinator for Biology"** (Function) fueled by **"The Bio-Graph"** (Data).
**Principle**: The Navigator is just *one app*. The Data is the *asset*.

## 1. Core Concept: "The Bio-Akinator" (Logic Engine)
A fast, logic-driven function for **Identification Support**.
*   **Goal**: Shortest path to ID.
*   **Method**: Binary Search using data from Tier 1 (Handbooks) & Tier 2 (Maps).
*   **UX**: "Is it shiny?" -> Yes -> "Is it near water?" -> Yes. -> "Result: Dace."

## 2. The Data Strategy: "Collect Everything Interesting"
We do NOT limit ingestion to *only* what the Akinator needs today.
We ingest **"All Interesting Context"** for future features.

*   **Observation Pages (Shu-List)**:
    *   Show "Rearing Methods" (from Insectarium).
    *   Show "Historical Names" (from Cultural Books).
    *   Show "Citations/Amazon Links" (Traffic Back to Authors).
*   **Future Apps**:
    *   "Bio-History Map" (using cultural data).
    *   "Breeding Assistant" (using rearing data).

## 2. Core Philosophy: Logic Over Vision
**"We do not use smartphone photos for identification."**
Unlike Google Lens or typical AI scanners, the Bio-Akinator does not rely on computer vision of user-uploaded photos.
*   **Input**: User observations (Answers to Yes/No questions).
*   **Engine**: Shortest-path logic derived from the Bio-Graph.
*   **Why**: To foster "observation methodology" rather than passive scanning.

## 3. Data Source: The Bio-Graph
*   **Structured Logic**: Diagnostic keys extracted from expert texts (e.g., Kuwagata Handbook).
*   **Visual Semantics**: Professional diagrams and photos from books are preserved for *output* (reference), not input.

## 3. The Functional Flow
1.  **Ingestion**: Agent mines *everything* (Logic, Time, Place, Culture).
2.  **Storage**: V3 Bio-Graph holds all connections.
3.  **Application 1 (Navigator)**: Pulls *only* Logic/Map data for the "Akinator Game".
4.  **Application 2 (Detail Page)**: Pulls *all* data to show the rich context + Purchase Links.

## 4. The "Strongest" Definition
The "Strongest Bio-Navigator" is the one that **never fails to identify**, and then **deeply educates** using the rich data (Culture/History) stored in the background.
It connects the "Game of ID" to the "Depth of Reality".
